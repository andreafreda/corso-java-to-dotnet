## Un po' di storia
Se usi Java moderno, probabilmente hai già apprezzato le switch expressions di Java 14 e il pattern matching di Java 16+. Buona notizia: in C# queste feature esistono da più tempo e sono più potenti. Se invece vieni da un Java più datato dove lo switch era solo una serie di `case` con `break`, preparati a cambiare modo di pensare — in meglio.

---

## Switch expression — addio `break`

In Java moderno hai già la sintassi con `->` che elimina il fall-through e restituisce un valore. C# funziona allo stesso modo:

```csharp
// C# — switch expression
string descrizione = stato switch
{
    Stato.Attivo    => "Il servizio è attivo",
    Stato.Inattivo  => "Il servizio è fermo",
    Stato.Sospeso   => "Il servizio è sospeso",
    _               => "Stato sconosciuto"   // _ è il default, come in Java
};
```

```java
// Java — per confronto
String descrizione = switch (stato) {
    case ATTIVO    -> "Il servizio è attivo";
    case INATTIVO  -> "Il servizio è fermo";
    case SOSPESO   -> "Il servizio è sospeso";
    default        -> "Stato sconosciuto";
};
```

Molto simile. La differenza di sintassi: in C# il soggetto viene prima (`stato switch { ... }`), in Java viene dopo il `switch` (`switch (stato) { ... }`). Questione di abitudine.

Il compilatore ti avvisa se non hai coperto tutti i casi di un enum — esattamente come in Java.

## Type pattern — controllare il tipo e fare il cast insieme

In Java dal 16+ hai:
```java
if (obj instanceof String s) {
    System.out.println(s.length()); // s è già String
}
```

In C# funziona allo stesso modo, e puoi usarlo anche dentro uno switch:

```csharp
// if con type pattern
if (notifica is EmailNotifica email)
{
    Console.WriteLine(email.Destinatario);
}

// switch con type pattern — molto più potente
string descrizione = notifica switch
{
    EmailNotifica e   => $"Email a {e.Destinatario}",
    SmsNotifica s     => $"SMS a {s.Numero}",
    PushNotifica p    => $"Push a {p.DeviceId}",
    null              => "Nessuna notifica",
    _                 => "Tipo sconosciuto"
};
```

Niente `as` + null check, niente cast esplicito — il pattern fa tutto in una riga.

---

## Property pattern — controllare le proprietà dell'oggetto

Questo non ha equivalente diretto in Java. Puoi fare match su valori specifici delle property di un oggetto direttamente nel pattern:

```csharp
string categoria = ordine switch
{
    { Totale: > 1000, Cliente: "VIP" }  => "Ordine VIP prioritario",
    { Totale: > 1000 }                  => "Ordine grande",
    { Totale: > 100 }                   => "Ordine medio",
    { Totale: 0 }                       => "Ordine vuoto",
    _                                   => "Ordine piccolo"
};
```

I pattern vengono valutati dall'alto verso il basso — il primo che fa match vince. Come in Java, l'ordine conta.

---

## Positional pattern — decostruzione

Funziona con i `record` (che hanno la decostruzione automatica):

```csharp
public record Punto(double X, double Y);

string quadrante = punto switch
{
    (> 0, > 0) => "Primo quadrante",
    (< 0, > 0) => "Secondo quadrante",
    (< 0, < 0) => "Terzo quadrante",
    (> 0, < 0) => "Quarto quadrante",
    _          => "Sull'asse"
};
```

---

## `when` — guardie sui pattern

Puoi aggiungere condizioni aggiuntive con `when` (equivalente di `when` nelle guard clause di Kotlin, o della `when` nei pattern Java):

```csharp
string messaggio = ordine switch
{
    { Totale: var t } when t > 1000 && t < 5000 => "Ordine grande",
    { Totale: var t } when t >= 5000            => "Ordine enterprise",
    { Cliente: var c } when c.StartsWith("VIP") => "Cliente VIP",
    _                                           => "Ordine normale"
};
```

## Pattern combinati con `and`, `or`, `not`

```csharp
string fascia = età switch
{
    < 18          => "Minorenne",
    >= 18 and < 65 => "Adulto",
    >= 65         => "Senior"
};

// not pattern
if (valore is not null)
    Console.WriteLine(valore);

// or pattern
if (codice is 400 or 401 or 403)
    Console.WriteLine("Errore client");
```

---

## Il pattern nell'`if` — `is`

Oltre allo switch, il pattern matching funziona anche negli `if` con l'operatore `is`:

```csharp
// Type pattern
if (obj is string testo && testo.Length > 0)
    Console.WriteLine(testo.ToUpper());

// Constant pattern
if (valore is 42)
    Console.WriteLine("La risposta!");

// Not null
if (risultato is not null)
    elabora(risultato);
```

---

## In sintesi

- Le **switch expressions** eliminano `break` e fall-through — stessa idea di Java 14+, sintassi speculare.
- Il **type pattern** (`is Tipo variabile`) combina controllo tipo e cast — come Java 16+, ma usabile anche dentro switch.
- Il **property pattern** (`{ Proprietà: valore }`) non ha equivalente Java e permette match su struttura dell'oggetto.
- Il **positional pattern** funziona con i record e permette decostruzione diretta nel match.
- Le **guardie `when`** aggiungono condizioni arbitrarie ai pattern.
- `and`, `or`, `not` compongono i pattern in modo leggibile.
