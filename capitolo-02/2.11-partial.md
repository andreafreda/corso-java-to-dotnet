# 2.11 Partial class e partial methods

> **Analogia Java**: nessun equivalente diretto — in Java il codice generato e il codice scritto a mano convivono nello stesso file, oppure si separano tramite ereditarietà

---

## Il concetto

In C# puoi spezzare la definizione di una classe su più file usando la keyword `partial`. Il compilatore li unisce tutti in un'unica classe durante la build — come se fossero un file solo.

```csharp
// File: Ordine.cs — scritto a mano
public partial class Ordine
{
    public void Valida()
    {
        if (Totale < 0) throw new ArgumentException("Totale non può essere negativo");
    }
}

// File: Ordine.Generated.cs — generato automaticamente (non toccare)
public partial class Ordine
{
    public int Id { get; set; }
    public string Cliente { get; set; } = string.Empty;
    public decimal Totale { get; set; }
}
```

Per il compilatore è una sola classe `Ordine` con tutti i membri di entrambi i file.

---

## Quando le incontri nella pratica

Le partial class non sono una feature che usi spesso nelle classi di business — il loro caso d'uso principale è la **code generation**. Le incontrerai in questi contesti:

**EF Core migrations** — i file di migrazione generati sono partial class. Non le modifichi, ma le vedi.

**Source Generators** — una feature di .NET 5+ che genera codice a compile time. Molte librerie moderne (come il nuovo `System.Text.Json` con `JsonSerializerContext`, o `LoggerMessage`) generano partial class che tu completi:

```csharp
// Tu scrivi questo:
[JsonSerializable(typeof(Ordine))]
public partial class OrdineJsonContext : JsonSerializerContext { }

// Il source generator aggiunge automaticamente l'implementazione in un file .Generated.cs
```

**WinForms / WPF / Blazor** — il designer genera il codice dell'interfaccia grafica in un file `.Designer.cs` o `.razor.cs` separato, tu scrivi la logica nell'altro. Se hai mai visto un file `Form1.Designer.cs` in un vecchio progetto Windows Forms, era una partial class.

**ASP.NET Core Razor Pages** — il file `.cshtml.cs` che scrivi tu è una partial class rispetto al codice generato dal compilatore Razor.

## Partial methods

Le partial class possono contenere **partial methods** — metodi dichiarati in una parte e implementati (opzionalmente) nell'altra:

```csharp
// File generato — dichiara il metodo senza implementazione
public partial class Ordine
{
    partial void OnTotaleChanged(decimal vecchioValore, decimal nuovoValore);

    public decimal Totale
    {
        get => _totale;
        set
        {
            var vecchio = _totale;
            _totale = value;
            OnTotaleChanged(vecchio, value); // chiamata al partial method
        }
    }
}

// File scritto a mano — implementa il metodo se serve
public partial class Ordine
{
    partial void OnTotaleChanged(decimal vecchioValore, decimal nuovoValore)
    {
        Console.WriteLine($"Totale cambiato da {vecchioValore} a {nuovoValore}");
    }
}
```

Se non implementi il partial method, il compilatore rimuove completamente la chiamata — zero overhead a runtime. È un hook opzionale pensato esattamente per la code generation: il generatore espone i punti di estensione, tu decidi quali implementare.

---

## La convenzione per i file generati

Nei progetti .NET troverai spesso file con il suffisso `.Generated.cs` o `.g.cs` — è la convenzione per indicare che quel file non va toccato a mano:

```
Ordine.cs              ← scrivi qui
Ordine.Generated.cs    ← generato automaticamente, non toccare
```

I file generati vengono spesso esclusi da `.gitignore` o da certi strumenti di analisi statica, proprio perché non sono codice "umano".

---

## In sintesi

- `partial` divide la definizione di una classe su più file — il compilatore li unisce.
- **Non è una feature per il codice di business** — il suo caso d'uso è quasi esclusivamente la code generation.
- La incontrerai con EF Core, Source Generators, WinForms/WPF, Razor Pages — quasi sempre in file `.Generated.cs` che non devi modificare.
- I **partial methods** sono hook opzionali: se non li implementi, il compilatore rimuove la chiamata senza lasciare overhead.
