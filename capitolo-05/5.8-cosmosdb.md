# 5.8 Cosmos DB con .NET: esempio pratico

---

## Introduzione

Azure Cosmos DB è il database NoSQL di Azure: documento, globalmente distribuito, latenza garantita sotto i 10ms. In .NET si usa con l'**Azure Cosmos DB SDK** — nessun ORM, lavori direttamente con documenti JSON. Questo paragrafo è un **esempio standalone** — non modifica `OrderService`.

---

## Modello documento

```csharp
// Infrastructure/Documents/OrderDocument.cs
// Il Cosmos SDK 3.x usa Newtonsoft.Json come serializer di default — non System.Text.Json
using Newtonsoft.Json;

namespace OrderService.Infrastructure.Documents;

public class OrderDocument
{
    [JsonProperty("id")]
    public string Id { get; set; } = Guid.NewGuid().ToString();

    [JsonProperty("customer")]
    public string Customer { get; set; } = string.Empty;

    [JsonProperty("total")]
    public decimal Total { get; set; }

    [JsonProperty("status")]
    public string Status { get; set; } = string.Empty;

    [JsonProperty("createdAt")]
    public DateTime CreatedAt { get; set; } = DateTime.UtcNow;

    // Partition key — distribuisce i dati tra le partizioni fisiche
    // "status" è una scelta didattica; in produzione usa un campo ad alta cardinalità
    [JsonProperty("partitionKey")]
    public string PartitionKey => Status;
}
```

## Repository Cosmos DB

```csharp
// Infrastructure/Repositories/CosmosOrderRepository.cs
using Microsoft.Azure.Cosmos;
using Microsoft.Azure.Cosmos.Linq;
using OrderService.Infrastructure.Documents;

namespace OrderService.Infrastructure.Repositories;

public class CosmosOrderRepository
{
    private readonly Container _container;

    public CosmosOrderRepository(CosmosClient client)
    {
        _container = client.GetContainer("OrderDb", "orders");
    }

    // CREATE
    public async Task<OrderDocument> AddAsync(OrderDocument doc, CancellationToken ct = default)
    {
        var response = await _container.CreateItemAsync(
            doc,
            new PartitionKey(doc.PartitionKey),
            cancellationToken: ct);
        return response.Resource;
    }

    // READ per id + partition key — O(1), la lettura più veloce
    public async Task<OrderDocument?> GetByIdAsync(string id, string partitionKey, CancellationToken ct = default)
    {
        try
        {
            var response = await _container.ReadItemAsync<OrderDocument>(
                id,
                new PartitionKey(partitionKey),
                cancellationToken: ct);
            return response.Resource;
        }
        catch (CosmosException ex) when (ex.StatusCode == System.Net.HttpStatusCode.NotFound)
        {
            return null;
        }
    }

    // READ con query LINQ
    public async Task<IEnumerable<OrderDocument>> GetByStatusAsync(string status, CancellationToken ct = default)
    {
        var iterator = _container
            .GetItemLinqQueryable<OrderDocument>()
            .Where(o => o.Status == status)
            .ToFeedIterator();

        var results = new List<OrderDocument>();
        while (iterator.HasMoreResults)
        {
            var page = await iterator.ReadNextAsync(ct);
            results.AddRange(page);
        }
        return results;
    }

    // UPDATE — inserisce se non esiste, aggiorna se esiste
    public async Task<OrderDocument> UpsertAsync(OrderDocument doc, CancellationToken ct = default)
    {
        var response = await _container.UpsertItemAsync(
            doc,
            new PartitionKey(doc.PartitionKey),
            cancellationToken: ct);
        return response.Resource;
    }

    // DELETE
    public async Task DeleteAsync(string id, string partitionKey, CancellationToken ct = default)
    {
        await _container.DeleteItemAsync<OrderDocument>(
            id,
            new PartitionKey(partitionKey),
            cancellationToken: ct);
    }
}
```

## Registrazione e inizializzazione

```csharp
// Program.cs
var endpoint = builder.Configuration["CosmosDb:Endpoint"]!;
var key      = builder.Configuration["CosmosDb:Key"]!;

// Il bypass SSL è necessario solo con l'emulatore locale — mai in produzione
var cosmosOptions = builder.Environment.IsDevelopment()
    ? new CosmosClientOptions
      {
          HttpClientFactory = () => new HttpClient(new HttpClientHandler
          {
              ServerCertificateCustomValidationCallback =
                  HttpClientHandler.DangerousAcceptAnyServerCertificateValidator
          }),
          ConnectionMode = ConnectionMode.Gateway
      }
    : new CosmosClientOptions();

builder.Services.AddSingleton(new CosmosClient(endpoint, key, cosmosOptions));
builder.Services.AddScoped<CosmosOrderRepository>();

// Inizializzazione database e container in Development
if (app.Environment.IsDevelopment())
{
    var cosmos = app.Services.GetRequiredService<CosmosClient>();
    var db = await cosmos.CreateDatabaseIfNotExistsAsync("OrderDb");
    await db.Database.CreateContainerIfNotExistsAsync("orders", "/partitionKey");
}
```

## Spiegazione

**Partition key** — ogni documento appartiene a una partizione logica. La scelta è critica per le performance: deve distribuire uniformemente i dati ed essere presente in tutte le query. `status` è una scelta didattica — ha bassa cardinalità e causerebbe hot partition in produzione.

**`ReadItemAsync` vs query** — `ReadItemAsync` con id e partition key è O(1). Usala sempre quando hai entrambi. Le query LINQ sono necessarie quando non hai la partition key o vuoi filtrare su altri campi.

**`UpsertItemAsync`** — inserisce se il documento non esiste, aggiorna se esiste. Equivalente di `MERGE` in SQL — utile per operazioni idempotenti.

**Nessuna migration** — Cosmos DB accetta qualsiasi documento JSON. Aggiungi campi senza migration, ma perdi la consistenza garantita dal database relazionale.

## Confronto con Java

| Java (Azure Cosmos SDK / Spring Data Cosmos) | .NET (Azure Cosmos SDK) |
|---|---|
| `CosmosAsyncClient` | `CosmosClient` |
| `container.createItem()` | `Container.CreateItemAsync()` |
| `container.readItem()` | `Container.ReadItemAsync()` |
| `container.upsertItem()` | `Container.UpsertItemAsync()` |
| `container.deleteItem()` | `Container.DeleteItemAsync()` |
| `CosmosRepository<T, ID>` (Spring Data) | Repository scritto manualmente |
| `@Container(containerName = "orders")` | Container passato via SDK |
