# 5.1 Azure SQL in locale con Docker

---

## Introduzione

Prima di toccare EF Core, serve un database. Usiamo **Azure SQL Edge** — un'immagine Docker di SQL Server ottimizzata per ambienti locali e ARM, ufficialmente supportata da Microsoft. In produzione userai Azure SQL su cloud; in locale è identico, cambia solo la stringa di connessione.

Se preferisci PostgreSQL, trovi la nota a fondo paragrafo — il resto del capitolo funziona identicamente, cambia solo il package NuGet e la stringa di connessione.

---

## Come si faceva in Java

```yaml
# docker-compose.yml con SQL Server per Spring Boot
services:
  db:
    image: mcr.microsoft.com/mssql/server:2022-latest
    environment:
      ACCEPT_EULA: "Y"
      SA_PASSWORD: "Password123!"
    ports:
      - "1433:1433"
```

```yaml
# application.yml
spring:
  datasource:
    url: jdbc:sqlserver://localhost:1433;databaseName=orderdb;encrypt=false
    username: sa
    password: Password123!
    driver-class-name: com.microsoft.sqlserver.jdbc.SQLServerDriver
```

---

## `docker-compose.yml` per Azure SQL Edge

Crea il file nella root della solution:

```
OrderService/
├── docker-compose.yml    ← nuovo
├── OrderService.sln
├── OrderService.Api/
├── OrderService.Core/
└── OrderService.Infrastructure/
```

```yaml
# docker-compose.yml
services:
  azuresql:
    image: mcr.microsoft.com/azure-sql-edge:latest
    environment:
      ACCEPT_EULA: "Y"
      MSSQL_SA_PASSWORD: "Password123!"
    ports:
      - "1433:1433"
```

```bash
# Avvia il database
docker compose up -d

# Verifica che sia in ascolto
docker compose ps
# azuresql   running   0.0.0.0:1433->1433/tcp
```

---

## Stringa di connessione in `appsettings.Development.json`

```json
{
  "ConnectionStrings": {
    "Default": "Server=localhost,1433;Database=OrderDb;User Id=sa;Password=Password123!;TrustServerCertificate=True"
  }
}
```

`TrustServerCertificate=True` è necessario in locale perché Azure SQL Edge usa un certificato self-signed. Non usarlo in produzione.

---

## Spiegazione

**Azure SQL Edge vs SQL Server** — Azure SQL Edge è un subset di SQL Server ottimizzato per edge computing. Per sviluppo locale è identico a SQL Server — stesse istruzioni T-SQL, stesso driver, stesso EF Core provider. L'immagine è più leggera e gira su ARM (Apple Silicon incluso).

**Perché non Azure SQL su cloud in sviluppo** — puoi farlo, ma hai latenza, costi, e dipendenza da una connessione internet. Docker in locale è più veloce e completamente offline.

**`MSSQL_SA_PASSWORD`** — la password dell'utente `sa` (system administrator). Deve rispettare la policy di complessità di SQL Server: almeno 8 caratteri, maiuscole, minuscole, numeri. `Password123!` soddisfa i requisiti per l'ambiente locale.

## Nota per chi usa PostgreSQL

Se preferisci PostgreSQL, il `docker-compose.yml` diventa:

```yaml
services:
  postgres:
    image: postgres:16
    environment:
      POSTGRES_DB: OrderDb
      POSTGRES_USER: user
      POSTGRES_PASSWORD: secret
    ports:
      - "5432:5432"
```

La stringa di connessione in `appsettings.Development.json`:

```json
{
  "ConnectionStrings": {
    "Default": "Host=localhost;Database=OrderDb;Username=user;Password=secret"
  }
}
```

Nel capitolo 5.2 aggiungerai il package EF Core corretto:
- Azure SQL → `dotnet add package Microsoft.EntityFrameworkCore.SqlServer`
- PostgreSQL → `dotnet add package Npgsql.EntityFrameworkCore.PostgreSQL`

Il resto del codice — `DbContext`, migrations, repository — è identico.

## Confronto con Java

| Spring Boot | ASP.NET Core |
|---|---|
| `docker-compose.yml` con PostgreSQL/MySQL | `docker-compose.yml` con Azure SQL Edge |
| `spring.datasource.url` in `application.yml` | `ConnectionStrings:Default` in `appsettings.Development.json` |
| Driver JDBC nel `pom.xml` | EF Core provider nel `.csproj` — aggiunto in 5.2 |
