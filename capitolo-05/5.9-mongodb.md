# 5.9 MongoDB con .NET: esempio pratico

---

## Introduzione

MongoDB è il database a documenti più diffuso — JSON nativo, schema flessibile, query potenti. In .NET si usa con il driver ufficiale **MongoDB.Driver**. Come il paragrafo Cosmos DB, questo è un **esempio standalone** — non modifica `OrderService`.

---

## Come si faceva in Java

```java
// Spring Data MongoDB
@Document(collection = "orders")
public class OrderDocument {
    @Id
    private String id;
    private String customer;
    private BigDecimal total;
    private String status;
    private LocalDateTime createdAt;
}

@Repository
public interface OrderMongoRepository extends MongoRepository<OrderDocument, String> {
    List<OrderDocument> findByStatus(String status);
    List<OrderDocument> findByTotalGreaterThan(BigDecimal min);
}
```

```java
// MongoTemplate per operazioni avanzate
@Autowired
private MongoTemplate mongoTemplate;

Query query = new Query(Criteria.where("status").is("PENDING")
    .and("total").gt(100));
List<OrderDocument> orders = mongoTemplate.find(query, OrderDocument.class);
```

## Modello documento

```csharp
// Infrastructure/Documents/OrderDocument.cs
using MongoDB.Bson;
using MongoDB.Bson.Serialization.Attributes;

namespace OrderService.Infrastructure.Documents;

public class OrderDocument
{
    [BsonId]
    [BsonRepresentation(BsonType.ObjectId)]
    public string? Id { get; set; }

    [BsonElement("customer")]
    public string Customer { get; set; } = string.Empty;

    [BsonElement("total")]
    public decimal Total { get; set; }

    [BsonElement("status")]
    public string Status { get; set; } = string.Empty;

    [BsonElement("createdAt")]
    public DateTime CreatedAt { get; set; } = DateTime.UtcNow;
}
```

`[BsonId]` marca la chiave primaria del documento — MongoDB genera automaticamente un `ObjectId` se non ne fornisci uno. `[BsonElement]` controlla il nome del campo nel documento JSON, come `@JsonPropertyName` in Cosmos DB.

## Repository MongoDB

```csharp
// Infrastructure/Repositories/MongoOrderRepository.cs
using MongoDB.Driver;
using OrderService.Infrastructure.Documents;

namespace OrderService.Infrastructure.Repositories;

public class MongoOrderRepository
{
    private readonly IMongoCollection<OrderDocument> _collection;

    public MongoOrderRepository(IMongoDatabase database)
    {
        _collection = database.GetCollection<OrderDocument>("orders");
    }

    // CREATE
    public async Task<OrderDocument> AddAsync(OrderDocument doc, CancellationToken ct = default)
    {
        await _collection.InsertOneAsync(doc, cancellationToken: ct);
        return doc;  // Id è popolato dopo InsertOneAsync
    }

    // READ per Id
    public async Task<OrderDocument?> GetByIdAsync(string id, CancellationToken ct = default)
    {
        var filter = Builders<OrderDocument>.Filter.Eq(o => o.Id, id);
        return await _collection.Find(filter).FirstOrDefaultAsync(ct);
    }

    // READ con filtro
    public async Task<IEnumerable<OrderDocument>> GetByStatusAsync(string status, CancellationToken ct = default)
    {
        var filter = Builders<OrderDocument>.Filter.Eq(o => o.Status, status);
        return await _collection.Find(filter).ToListAsync(ct);
    }

    // READ con filtro multiplo e ordinamento
    public async Task<IEnumerable<OrderDocument>> GetRecentPendingAsync(decimal minTotal, CancellationToken ct = default)
    {
        var filter = Builders<OrderDocument>.Filter.And(
            Builders<OrderDocument>.Filter.Eq(o => o.Status, "Pending"),
            Builders<OrderDocument>.Filter.Gte(o => o.Total, minTotal));

        var sort = Builders<OrderDocument>.Sort.Descending(o => o.CreatedAt);

        return await _collection.Find(filter).Sort(sort).ToListAsync(ct);
    }

    // UPDATE
    public async Task UpdateStatusAsync(string id, string newStatus, CancellationToken ct = default)
    {
        var filter = Builders<OrderDocument>.Filter.Eq(o => o.Id, id);
        var update = Builders<OrderDocument>.Update
            .Set(o => o.Status, newStatus);

        await _collection.UpdateOneAsync(filter, update, cancellationToken: ct);
    }

    // DELETE
    public async Task DeleteAsync(string id, CancellationToken ct = default)
    {
        var filter = Builders<OrderDocument>.Filter.Eq(o => o.Id, id);
        await _collection.DeleteOneAsync(filter, ct);
    }
}
```

## Registrazione e configurazione

```csharp
// Program.cs
var mongoSettings = builder.Configuration.GetSection("MongoDb");
builder.Services.AddSingleton<IMongoClient>(
    new MongoClient(mongoSettings["ConnectionString"]));
builder.Services.AddScoped<IMongoDatabase>(sp =>
    sp.GetRequiredService<IMongoClient>().GetDatabase(mongoSettings["Database"]));
builder.Services.AddScoped<MongoOrderRepository>();
```

## Spiegazione

**`Builders<T>.Filter`** — MongoDB.Driver usa un builder per costruire filtri, update e sort in modo type-safe. È più verboso di LINQ ma mappa direttamente sull'API MongoDB — ogni operazione corrisponde a un operatore MongoDB (`$eq`, `$gte`, `$and`, `$set`).

**`InsertOneAsync` popola l'Id** — dopo `InsertOneAsync`, il campo `Id` del documento è popolato con l'`ObjectId` generato da MongoDB. Non serve una query separata per recuperarlo.

**`UpdateOneAsync` con `$set`** — aggiorna solo i campi specificati, non sostituisce l'intero documento. Equivalente di `UPDATE SET` in SQL, a differenza di `ReplaceOneAsync` che sostituisce il documento intero.

**Schema flessibile** — puoi aggiungere campi al documento senza migration. I documenti esistenti semplicemente non avranno quel campo — gestisci il default in C# con `?` e valori di default.

## Confronto con Java

| Java (Spring Data MongoDB / Driver) | .NET (MongoDB.Driver) |
|---|---|
| `@Document(collection = "orders")` | `[BsonId]`, `[BsonElement]` sul modello |
| `MongoRepository<T, ID>` | Repository scritto manualmente |
| `MongoTemplate.find(query, T.class)` | `collection.Find(filter).ToListAsync()` |
| `Criteria.where("field").is(value)` | `Builders<T>.Filter.Eq(o => o.Field, value)` |
| `Update.set("field", value)` | `Builders<T>.Update.Set(o => o.Field, value)` |
| `mongoTemplate.insert(doc)` | `collection.InsertOneAsync(doc)` |
| `mongoTemplate.remove(query, T.class)` | `collection.DeleteOneAsync(filter)` |
