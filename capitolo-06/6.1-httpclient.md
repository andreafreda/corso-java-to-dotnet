# 6.1 HttpClient e IHttpClientFactory

> **Analogia Java**: equivalente di `RestTemplate` / `WebClient`

---

## Introduzione

`OrderService` deve chiamare un altro microservizio ŌĆö `InventoryService` ŌĆö per verificare la disponibilit├Ā prima di confermare un ordine. In Java usavi `RestTemplate` (sincrono) o `WebClient` (reattivo). In .NET il client HTTP ├© `HttpClient`, ma non va istanziato direttamente: la gestione del ciclo di vita ├© la trappola pi├╣ comune per i developer .NET. La soluzione ├© `IHttpClientFactory`.

---

## Come si faceva in Java

```java
// RestTemplate ŌĆö il modo classico
@Service
public class InventoryClient {

    private final RestTemplate restTemplate;

    public InventoryClient(RestTemplateBuilder builder) {
        this.restTemplate = builder
            .rootUri("http://inventory-service")
            .build();
    }

    public boolean isAvailable(int productId, int quantity) {
        return restTemplate.getForObject(
            "/inventory/{id}/available?quantity={qty}",
            Boolean.class,
            productId, quantity);
    }
}
```

```java
// WebClient ŌĆö il modo reattivo/moderno
@Service
public class InventoryClient {

    private final WebClient webClient;

    public InventoryClient(WebClient.Builder builder) {
        this.webClient = builder
            .baseUrl("http://inventory-service")
            .build();
    }

    public Mono<Boolean> isAvailable(int productId, int quantity) {
        return webClient.get()
            .uri("/inventory/{id}/available?quantity={qty}", productId, quantity)
            .retrieve()
            .bodyToMono(Boolean.class);
    }
}
```

## Il problema con `new HttpClient()`

```csharp
// ŌØī MAI fare cos├¼ ŌĆö esaurisce le connessioni TCP disponibili
public async Task<bool> IsAvailableAsync(int productId, int quantity)
{
    using var client = new HttpClient();  // crea e distrugge una connessione a ogni chiamata
    var response = await client.GetAsync($"http://inventory-service/inventory/{productId}/available?quantity={quantity}");
    return response.IsSuccessStatusCode;
}
```

`HttpClient` mantiene le connessioni TCP aperte internamente. Creare e distruggere istanze con `using` non chiude le connessioni immediatamente ŌĆö le lascia in stato `TIME_WAIT`, esaurendo i socket disponibili sotto carico. Questo ├© il **socket exhaustion**, il bug di produzione pi├╣ comune con `HttpClient`.

---

## `IHttpClientFactory` ŌĆö la soluzione corretta

> ­¤ōØ **Code Step 12** ŌĆö Aggiungi `InventoryClient` e registra `IHttpClientFactory`

`IHttpClientFactory` gestisce un pool di connessioni `HttpMessageHandler` ŌĆö le connessioni vengono riusate correttamente e ruotate periodicamente per rispettare i cambiamenti DNS.

```csharp
// Infrastructure/Http/InventoryClient.cs
using System.Net.Http.Json;
using Microsoft.Extensions.Logging;
using OrderService.Core.Interfaces;

namespace OrderService.Infrastructure.Http;

public class InventoryClient : IInventoryClient
{
    private readonly HttpClient _client;
    private readonly ILogger<InventoryClient> _logger;

    // HttpClient viene iniettato gi├Ā configurato da IHttpClientFactory ŌĆö nessuna stringa magica
    public InventoryClient(HttpClient client, ILogger<InventoryClient> logger)
    {
        _client = client;
        _logger = logger;
    }

    public async Task<bool> IsAvailableAsync(int productId, int quantity, CancellationToken ct = default)
    {
        try
        {
            var response = await _client.GetAsync(
                $"/inventory/{productId}/available?quantity={quantity}", ct);

            response.EnsureSuccessStatusCode();
            return await response.Content.ReadFromJsonAsync<bool>(ct);
        }
        catch (HttpRequestException ex)
        {
            _logger.LogWarning(ex, "InventoryService non raggiungibile per productId={ProductId}", productId);
            return false;  // fallback: assume non disponibile
        }
    }
}
```

## Named client ŌĆö quando serve

Il typed client ├© il pattern standard. Il **named client** (`factory.CreateClient("nome")`) ├© utile in casi specifici: quando devi creare pi├╣ istanze dello stesso tipo con configurazioni diverse, o quando il client ├© una classe di terze parti che non puoi modificare.

```csharp
// Named client ŌĆö utile quando servono pi├╣ configurazioni per lo stesso tipo
services.AddHttpClient("inventory-primary", client =>
    client.BaseAddress = new Uri("http://inventory-service-1"));

services.AddHttpClient("inventory-secondary", client =>
    client.BaseAddress = new Uri("http://inventory-service-2"));

// Uso ŌĆö devi risolvere IHttpClientFactory esplicitamente
public class InventoryClient
{
    public InventoryClient(IHttpClientFactory factory)
    {
        _primary   = factory.CreateClient("inventory-primary");
        _secondary = factory.CreateClient("inventory-secondary");
    }
}
```

Per `OrderService` il typed client ├© la scelta corretta: un solo `InventoryClient`, una sola configurazione.

## Spiegazione

**Socket exhaustion** ŌĆö il problema di `new HttpClient()` non ├© ovvio. Il `using` chiama `Dispose()` ma le connessioni TCP restano in `TIME_WAIT` per ~240 secondi. Sotto carico esaurisci tutti i socket disponibili del processo. `IHttpClientFactory` riusa i `HttpMessageHandler` nel pool ŌĆö le connessioni vengono mantenute aperte correttamente.

**DNS refresh** ŌĆö `IHttpClientFactory` ruota i `HttpMessageHandler` ogni 2 minuti per default. Questo garantisce che i cambiamenti DNS (essenziali in Kubernetes e nei deployment rolling) vengano rispettati. Con una singola istanza `HttpClient` statica il DNS verrebbe cachato indefinitamente.

**`ReadFromJsonAsync`** ŌĆö metodo di estensione di `System.Net.Http.Json` (incluso in .NET 6+). Deserializza il body JSON direttamente sul tipo C# senza `JsonSerializer.Deserialize` esplicito.

## Confronto con Java

| Spring Boot | ASP.NET Core |
|---|---|
| `RestTemplate` | `HttpClient` via `IHttpClientFactory` |
| `RestTemplateBuilder.rootUri()` | `AddHttpClient("name", c => c.BaseAddress = ...)` |
| `WebClient.Builder.baseUrl()` | Typed client con `AddHttpClient<T>()` |
| `@LoadBalanced RestTemplate` | `IHttpClientFactory` + service discovery (6.2) |
| `restTemplate.getForObject(url, T.class)` | `client.GetFromJsonAsync<T>(url)` |
| `restTemplate.postForObject(url, body, T.class)` | `client.PostAsJsonAsync(url, body)` |
| `WebClientResponseException` | `HttpRequestException` |
