# 7.4 Implementare un AuthServer minimale in .NET

> **Analogia Java**: equivalente di Spring Authorization Server

---

## Introduzione

`OrderService` valida i JWT ma non li genera — è un **resource server**. Serve un **authorization server** separato che gestisce login, generazione token e refresh. In Java usavi Spring Authorization Server o Keycloak. In .NET la scelta più comune in produzione è **Azure Active Directory** o **OpenIddict**. In questo paragrafo implementiamo un AuthServer minimale in ASP.NET Core — senza librerie esterne — per capire il meccanismo prima di delegarlo a un provider esterno.

## Come si faceva in Java

```java
// Spring Authorization Server — configurazione
@Configuration
@EnableAuthorizationServer
public class AuthServerConfig extends AuthorizationServerConfigurerAdapter {

    @Override
    public void configure(ClientDetailsServiceConfigurer clients) throws Exception {
        clients.inMemory()
            .withClient("order-service-client")
            .secret("{bcrypt}" + passwordEncoder.encode("secret"))
            .authorizedGrantTypes("password", "refresh_token")
            .scopes("read", "write");
    }
}
```

## L'AuthServer minimale

> 📝 **Code Step 15** — Aggiungi endpoint `/auth/login` e `/auth/refresh` a `OrderService.Api`

In un sistema reale l'AuthServer è un servizio separato. Per il corso lo aggiungiamo direttamente in `OrderService.Api` — il meccanismo è identico.

### `AuthService`

```csharp
// Core/Services/AuthService.cs
using System.Security.Cryptography;
using System.Text;
using OrderService.Core.Entities;
using OrderService.Core.Interfaces;

namespace OrderService.Core.Services;

public class AuthService
{
    private readonly TokenService       _tokenService;
    private readonly IRefreshTokenStore _refreshStore;  // interfaccia — non implementazione

    private static readonly List<AppUser> Users =
    [
        new() { Id = "1", Email = "admin@example.com",
                PasswordHash = HashPassword("admin123"), Role = "Admin" },
        new() { Id = "2", Email = "user@example.com",
                PasswordHash = HashPassword("user123"),  Role = "User"  }
    ];

    public AuthService(TokenService tokenService, IRefreshTokenStore refreshStore)
    {
        _tokenService = tokenService;
        _refreshStore = refreshStore;
    }

    public TokenResponse? Login(string email, string password)
    {
        var user = Users.FirstOrDefault(u =>
            u.Email == email && u.PasswordHash == HashPassword(password));

        if (user is null) return null;

        return IssueTokens(user);
    }

    public TokenResponse? Refresh(string refreshToken)
    {
        var stored = _refreshStore.Get(refreshToken);
        if (stored is null || !stored.IsValid) return null;

        var user = Users.FirstOrDefault(u => u.Id == stored.UserId);
        if (user is null) return null;

        _refreshStore.Revoke(refreshToken);
        return IssueTokens(user);
    }

    private TokenResponse IssueTokens(AppUser user)
    {
        var accessToken  = _tokenService.GenerateToken(user.Id, user.Email, user.Role);
        var refreshToken = new RefreshToken
        {
            Token     = _tokenService.GenerateRefreshToken(),
            UserId    = user.Id,
            ExpiresAt = DateTime.UtcNow.AddDays(30),
            Revoked   = false
        };

        _refreshStore.Store(refreshToken);
        return new TokenResponse(accessToken, refreshToken.Token, ExpiresIn: 3600);
    }

    private static string HashPassword(string password)
    {
        var bytes = SHA256.HashData(Encoding.UTF8.GetBytes(password));
        return Convert.ToHexString(bytes);
    }
}
```

public string GenerateRefreshToken()

### Controller Auth

```csharp
// Api/Controllers/AuthController.cs
using Microsoft.AspNetCore.Mvc;
using OrderService.Api.Models.Auth;
using OrderService.Core.Services;

namespace OrderService.Api.Controllers;

[ApiController]
[Route("auth")]
public class AuthController : ControllerBase
{
    private readonly AuthService _authService;

    public AuthController(AuthService authService)
    {
        _authService = authService;
    }

    [HttpPost("login")]
    public IActionResult Login(LoginRequest request)
    {
        var result = _authService.Login(request.Email, request.Password);
        if (result is null) return Unauthorized(new { message = "Credenziali non valide" });
        return Ok(result);
    }

    [HttpPost("refresh")]
    public IActionResult Refresh([FromBody] string refreshToken)
    {
        var result = _authService.Refresh(refreshToken);
        if (result is null) return Unauthorized(new { message = "Refresh token non valido o scaduto" });
        return Ok(result);
    }
}
```

## Spiegazione

**SHA256 per le password** — nel corso usiamo SHA256 per semplicità. In produzione usa **BCrypt** o **Argon2** — algoritmi progettati per essere lenti e resistenti al brute force. .NET fornisce `BCrypt` tramite il package `BCrypt.Net-Next`.

**Refresh token come stringa opaca** — il refresh token non è un JWT ma una stringa casuale. Il suo significato sta nel database (o nel dictionary in-memory del corso) — non nel token stesso. Questo permette la revoca esplicita.

**`RefreshTokenStore` statico** — è un anti-pattern in produzione: i dati si perdono al riavvio e non scala orizzontalmente. In produzione salva i refresh token nel database con scadenza, utente associato e flag `revoked`.

**AuthServer separato in produzione** — in un'architettura microservizi l'AuthServer è un servizio dedicato. `OrderService` e gli altri servizi sono resource server — validano il token ma non lo generano. In Azure si usa **Azure Active Directory B2C** o **Entra ID** come AuthServer esterno.

| Spring Authorization Server / JJWT | ASP.NET Core |
|---|---|
| `AuthorizationServerConfigurerAdapter` | Controller `AuthController` custom |
| `authManager.authenticate()` | Logica di verifica manuale in `AuthService` |
| `UserDetailsService.loadUserByUsername()` | `Users.FirstOrDefault(u => u.Email == email)` |
| `passwordEncoder.matches()` | `HashPassword()` — BCrypt in produzione |
| `Jwts.builder().compact()` | `TokenService.GenerateToken()` |
| Token store in-memory / Redis | `RefreshTokenStore` (in-memory) / DB (produzione) |
| Keycloak come AuthServer esterno | Azure AD / Entra ID come AuthServer esterno |
