# 3.6 Configurazione: `appsettings.json` e Options pattern

> **Analogia Java**: equivalente di `application.yml` + `@ConfigurationProperties`

---

## Introduzione

Ogni applicazione ha bisogno di configurazione esterna: stringhe di connessione, timeout, URL di servizi esterni, feature flag. In ASP.NET Core questa configurazione vive in `appsettings.json` — un file JSON gerarchico che si comporta esattamente come `application.yml` in Spring Boot, con override per ambiente e supporto per variabili d'ambiente.

---

## Come si faceva in Java

```yaml
# application.yml
spring:
  datasource:
    url: jdbc:postgresql://localhost:5432/orderdb
    username: user
    password: secret

ordine:
  max-per-pagina: 50
  timeout-seconds: 30
```

```yaml
# application-dev.yml — override per sviluppo
spring:
  datasource:
    url: jdbc:postgresql://localhost:5432/orderdb_dev
```

```java
// Bind su POJO con @ConfigurationProperties
@ConfigurationProperties(prefix = "ordine")
@Component
public class OrdineProperties {
    private int maxPerPagina;
    private int timeoutSeconds;
    // getter e setter...
}

// Valore singolo con @Value
@Value("${ordine.max-per-pagina}")
private int maxPerPagina;
```

## `appsettings.json`

```json
{
  "ConnectionStrings": {
    "Default": "Host=localhost;Database=orderdb;Username=user;Password=secret"
  },
  "OrdineOptions": {
    "MaxPerPagina": 50,
    "TimeoutSeconds": 30
  },
  "Logging": {
    "LogLevel": {
      "Default": "Information",
      "Microsoft.AspNetCore": "Warning"
    }
  }
}
```

```json
// appsettings.Development.json — override per sviluppo
{
  "ConnectionStrings": {
    "Default": "Host=localhost;Database=orderdb_dev;Username=user;Password=secret"
  },
  "Logging": {
    "LogLevel": {
      "Default": "Debug"
    }
  }
}
```

## Options pattern

```csharp
// OrdineOptions.cs — POCO senza attributi
public class OrdineOptions
{
    public const string SectionName = "OrdineOptions";

    public int MaxPerPagina { get; set; } = 20;
    public int TimeoutSeconds { get; set; } = 30;
}
```

```csharp
// Program.cs — registrazione
builder.Services.Configure<OrdineOptions>(
    builder.Configuration.GetSection(OrdineOptions.SectionName));
```

```csharp
// OrdineService.cs — iniezione
public class OrdineService : IOrdineService
{
    private readonly OrdineOptions _options;

    public OrdineService(IOptions<OrdineOptions> options)
    {
        _options = options.Value;
    }

    public async Task<IEnumerable<OrdineDto>> GetAllAsync(int pagina)
    {
        var dimensione = Math.Min(pagina, _options.MaxPerPagina);
        // ...
    }
}
```

## Spiegazione

**`appsettings.{Environment}.json`** viene caricato automaticamente in base alla variabile d'ambiente `ASPNETCORE_ENVIRONMENT`. I valori sovrascrivono quelli del file base — esattamente come i profili Spring.

**Le tre varianti di `IOptions`** — questa è la parte che non ha un equivalente diretto in Spring e vale la pena capire bene:

| Tipo | Quando si aggiorna | Quando usarlo |
|---|---|---|
| `IOptions<T>` | Mai — singleton, legge una volta all'avvio | La maggior parte dei casi |
| `IOptionsSnapshot<T>` | Ad ogni richiesta HTTP | Configurazione che cambia tra deploy, in scenari Scoped |
| `IOptionsMonitor<T>` | In tempo reale, senza riavvio | Feature flag, configurazione live |

```csharp
// IOptionsMonitor — si aggiorna senza riavviare l'app
public class FeatureFlagService
{
    private readonly IOptionsMonitor<FeatureOptions> _options;

    public FeatureFlagService(IOptionsMonitor<FeatureOptions> options)
    {
        _options = options;
        // callback chiamato quando la configurazione cambia
        _options.OnChange(opts => Console.WriteLine("Configurazione aggiornata"));
    }

    public bool IsAbilitato(string feature) => _options.CurrentValue.Features[feature];
}
```

**Le variabili d'ambiente sovrascrivono il JSON** — e questo è fondamentale in Docker e Kubernetes. La chiave gerarchica `OrdineOptions:MaxPerPagina` diventa la variabile d'ambiente `OrdineOptions__MaxPerPagina` (doppio underscore come separatore):

```bash
# docker-compose.yml
environment:
  - ASPNETCORE_ENVIRONMENT=Production
  - OrdineOptions__MaxPerPagina=100
  - ConnectionStrings__Default=Host=db;Database=orderdb;...
```

Nessuna modifica al codice — la configurazione viene sovrascritta automaticamente.

**Leggere valori singoli** — equivalente di `@Value` in Spring, ma sconsigliato per le Options class:

```csharp
// Sconsigliato — usa Options pattern quando possibile
string connStr = builder.Configuration.GetConnectionString("Default");
string valore = builder.Configuration["OrdineOptions:MaxPerPagina"];

// Sicuro — lancia eccezione se la sezione non esiste
var section = builder.Configuration.GetRequiredSection("OrdineOptions");
```

## Confronto con Java

| Spring Boot | ASP.NET Core |
|---|---|
| `application.yml` | `appsettings.json` |
| `application-dev.yml` | `appsettings.Development.json` |
| `spring.profiles.active=dev` | `ASPNETCORE_ENVIRONMENT=Development` |
| `@ConfigurationProperties(prefix="ordine")` | `Configure<OrdineOptions>(config.GetSection("OrdineOptions"))` |
| `@Value("${ordine.max-per-pagina}")` | `IConfiguration["OrdineOptions:MaxPerPagina"]` |
| Singleton `@ConfigurationProperties` | `IOptions<T>` |
| `@RefreshScope` (Spring Cloud) | `IOptionsMonitor<T>` |
| `${ENV_VAR}` in YAML | Variabile d'ambiente sovrascrive automaticamente |
