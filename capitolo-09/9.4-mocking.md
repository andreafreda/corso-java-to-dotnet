# 9.4 Mocking con Moq o NSubstitute

> **Analogia Java**: equivalente di Mockito

---

## Introduzione

I test unitari isolano il codice sotto test dalle sue dipendenze. In Java usavi Mockito — `@Mock`, `when().thenReturn()`, `verify()`. In .NET hai due opzioni principali: **Moq** (il più diffuso) e **NSubstitute** (API più concisa). Entrambi fanno le stesse cose — la scelta è di stile.

---

## Come si faceva in Java

```java
// Mockito
@ExtendWith(MockitoExtension.class)
class OrderServiceTest {

    @Mock
    OrderRepository repository;

    @InjectMocks
    OrderService service;

    @Test
    void shouldSaveOrder() {
        // Stub
        when(repository.save(any())).thenReturn(new Order(1L, "Mario", 100.0));

        // Act
        var result = service.create(new CreateOrderRequest("Mario", 100.0));

        // Verify
        verify(repository, times(1)).save(argThat(o -> o.getCustomer().equals("Mario")));
        assertThat(result.getId()).isEqualTo(1L);
    }

    @Test
    void shouldHandleRepositoryException() {
        when(repository.save(any())).thenThrow(new RuntimeException("DB down"));
        assertThrows(RuntimeException.class, () -> service.create(new CreateOrderRequest("Mario", 100.0)));
    }
}
```

## Moq — il più diffuso

```bash
dotnet add OrderService.Tests package Moq
```

### Setup di base

```csharp
using Moq;

var repositoryMock = new Mock<IOrderRepository>();

// Setup — definisce il comportamento
repositoryMock
    .Setup(r => r.GetByIdAsync(1, It.IsAny<CancellationToken>()))
    .ReturnsAsync(new Order { Id = 1, Customer = "Mario Rossi" });

// Usa il mock
var repository = repositoryMock.Object;
var result     = await repository.GetByIdAsync(1);

// Verify — verifica che il metodo sia stato chiamato
repositoryMock.Verify(
    r => r.GetByIdAsync(1, It.IsAny<CancellationToken>()),
    Times.Once);
```

### Matchers

```csharp
// It.IsAny<T>() — qualsiasi valore del tipo
repositoryMock
    .Setup(r => r.AddAsync(It.IsAny<Order>(), It.IsAny<CancellationToken>()))
    .ReturnsAsync(new Order());

// It.Is<T>(predicate) — valore che soddisfa la condizione
repositoryMock
    .Setup(r => r.AddAsync(
        It.Is<Order>(o => o.Customer == "Mario Rossi"),
        It.IsAny<CancellationToken>()))
    .ReturnsAsync(new Order { Id = 1 });

// Verify con matcher
repositoryMock.Verify(
    r => r.AddAsync(
        It.Is<Order>(o => o.Total > 0 && o.Status == OrderStatus.Pending),
        It.IsAny<CancellationToken>()),
    Times.Once);
```

### Eccezioni e callback

```csharp
// Simula un'eccezione
repositoryMock
    .Setup(r => r.AddAsync(It.IsAny<Order>(), default))
    .ThrowsAsync(new InvalidOperationException("DB non disponibile"));

// Callback — esegui codice quando il metodo viene chiamato
repositoryMock
    .Setup(r => r.AddAsync(It.IsAny<Order>(), default))
    .Callback<Order, CancellationToken>((order, _) =>
    {
        order.Id = 42;  // simula l'assegnazione dell'Id dal DB
    })
    .ReturnsAsync((Order o, CancellationToken _) => o);
```

### `Mock.Of<T>()` — mock senza behavior

```csharp
// Per dipendenze che non ti interessano nel test corrente
var logger = Mock.Of<ILogger<OrderService>>();  // non verifica nulla, non fa nulla
var sut    = new OrderService(repositoryMock.Object, logger);
```

## NSubstitute — API più concisa

```bash
dotnet add OrderService.Tests package NSubstitute
```

```csharp
using NSubstitute;

// Creazione
var repository = Substitute.For<IOrderRepository>();

// Setup — più conciso di Moq
repository.GetByIdAsync(1, Arg.Any<CancellationToken>())
          .Returns(new Order { Id = 1, Customer = "Mario Rossi" });

// Eccezione
repository.AddAsync(Arg.Any<Order>(), default)
          .Throws(new InvalidOperationException("DB down"));

// Verify
await repository.Received(1).GetByIdAsync(1, Arg.Any<CancellationToken>());
await repository.DidNotReceive().AddAsync(Arg.Any<Order>(), default);
```

### Confronto Moq vs NSubstitute

```csharp
// Moq
mock.Setup(r => r.GetByIdAsync(1, It.IsAny<CancellationToken>()))
    .ReturnsAsync(order);
mock.Verify(r => r.GetByIdAsync(1, It.IsAny<CancellationToken>()), Times.Once);

// NSubstitute — equivalente
sub.GetByIdAsync(1, Arg.Any<CancellationToken>()).Returns(order);
await sub.Received(1).GetByIdAsync(1, Arg.Any<CancellationToken>());
```

NSubstitute è più conciso: lo stesso oggetto `sub` è sia stub che spy — non serve `.Object`.

## Fake vs Mock — quando usare `InMemoryOrderRepository`

```csharp
// Un fake è un'implementazione reale semplificata — non un mock
// InMemoryOrderRepository (cap. 4) è un fake eccellente per i test
public class InMemoryOrderRepository : IOrderRepository
{
    private readonly List<Order> _store = [];
    private int _nextId = 1;

    public Task<List<Order>> GetAllAsync(CancellationToken ct = default) =>
        Task.FromResult(_store.ToList());

    public Task<Order?> GetByIdAsync(int id, CancellationToken ct = default) =>
        Task.FromResult(_store.FirstOrDefault(o => o.Id == id));

    public Task<Order> AddAsync(Order order, CancellationToken ct = default)
    {
        order.Id = _nextId++;
        _store.Add(order);
        return Task.FromResult(order);
    }

    public Task<Order> UpdateAsync(Order order, CancellationToken ct = default)
    {
        var idx = _store.FindIndex(o => o.Id == order.Id);
        if (idx >= 0) _store[idx] = order;
        return Task.FromResult(order);
    }

    public Task DeleteAsync(int id, CancellationToken ct = default)
    {
        _store.RemoveAll(o => o.Id == id);
        return Task.CompletedTask;
    }
}

// Uso nei test — più semplice di un mock per scenari complessi
public class OrderServiceTests
{
    [Fact]
    public async Task CreateAsync_MultipleOrders_AssignsUniqueIds()
    {
        var repository = new InMemoryOrderRepository();  // fake, non mock
        var sut        = new OrderService(repository, Mock.Of<ILogger<OrderService>>());

        var order1 = await sut.CreateAsync(new CreateOrderRequest("Mario", 100m));
        var order2 = await sut.CreateAsync(new CreateOrderRequest("Giulia", 200m));

        order1.Id.Should().NotBe(order2.Id);
    }
}
```

## Spiegazione

**Mock vs Stub vs Spy vs Fake** — la terminologia è spesso confusa:
- **Stub**: restituisce valori predefiniti (`ReturnsAsync`)
- **Mock**: stub + verifica delle chiamate (`Verify`)
- **Spy**: oggetto reale con alcune chiamate intercettate
- **Fake**: implementazione reale semplificata (`InMemoryOrderRepository`)

Moq crea oggetti che sono sia stub che mock. Usa `Verify` solo per i comportamenti rilevanti per il test — non verificare ogni chiamata.

**`Times.Once` vs `Times.AtLeastOnce`** — se il tuo test verifica che un ordine viene salvato, usa `Times.Once` — non vuoi che venga salvato due volte accidentalmente. `Times.AtLeastOnce` è meno preciso e nasconde bug.

## Confronto con Java

| Mockito | Moq | NSubstitute |
|---|---|---|
| `@Mock` + `@InjectMocks` | `new Mock<T>()` + costruttore manuale | `Substitute.For<T>()` |
| `when(x.method()).thenReturn(y)` | `mock.Setup(x => x.Method()).Returns(y)` | `sub.Method().Returns(y)` |
| `any()` / `anyInt()` | `It.IsAny<T>()` | `Arg.Any<T>()` |
| `argThat(predicate)` | `It.Is<T>(predicate)` | `Arg.Is<T>(predicate)` |
| `verify(mock, times(1)).method()` | `mock.Verify(x => x.Method(), Times.Once)` | `sub.Received(1).Method()` |
| `verify(mock, never()).method()` | `mock.Verify(x => x.Method(), Times.Never)` | `sub.DidNotReceive().Method()` |
| `thenThrow(new Exception())` | `.Throws(new Exception())` | `.Throws(new Exception())` |
| `doAnswer(invocation -> ...)` | `.Callback<T>((arg) => ...)` | `.AndDoes(call => ...)` |
