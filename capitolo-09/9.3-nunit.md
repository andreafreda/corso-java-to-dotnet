# 9.3 NUnit in pratica: `[Test]`, `[TestCase]` e `[SetUp]`

> **Analogia Java**: più simile a JUnit 4 nello stile

---

## Introduzione

NUnit è il framework più vicino allo stile JUnit — `[SetUp]`, `[TearDown]`, `[TestCase]` sono diretti equivalenti di `@BeforeEach`, `@AfterEach`, `@ValueSource`. Se vieni da JUnit e vuoi la transizione più lineare, NUnit è la scelta naturale.

---

## Setup

```bash
dotnet new nunit -n OrderService.NUnitTests
dotnet add OrderService.NUnitTests reference OrderService.Core
dotnet add OrderService.NUnitTests package Moq
dotnet add OrderService.NUnitTests package FluentAssertions
```

## `[Test]` e `[SetUp]`

```csharp
// NUnit/Services/OrderServiceTests.cs
using FluentAssertions;
using Moq;
using NUnit.Framework;
using OrderService.Core.Entities;
using OrderService.Core.Interfaces.Repositories;
using OrderService.Core.Services;

namespace OrderService.NUnitTests.Services;

[TestFixture]
public class OrderServiceTests
{
    private Mock<IOrderRepository> _repositoryMock;
    private OrderService           _sut;

    [SetUp]   // ← equivalente di @BeforeEach
    public void Setup()
    {
        _repositoryMock = new Mock<IOrderRepository>();
        _sut            = new OrderService(_repositoryMock.Object, Mock.Of<ILogger<OrderService>>());
    }

    [TearDown]  // ← equivalente di @AfterEach — opzionale
    public void Cleanup()
    {
        // Cleanup se necessario
    }

    [Test]
    public async Task CreateAsync_ValidRequest_ReturnsCreatedOrder()
    {
        // Arrange
        var request  = new CreateOrderRequest("Mario Rossi", 150.00m);
        var expected = new Order { Id = 1, Customer = "Mario Rossi", Total = 150.00m };

        _repositoryMock
            .Setup(r => r.AddAsync(It.IsAny<Order>(), It.IsAny<CancellationToken>()))
            .ReturnsAsync(expected);

        // Act
        var result = await _sut.CreateAsync(request);

        // Assert
        result.Should().NotBeNull();
        result.Customer.Should().Be("Mario Rossi");
    }

    [Test]
    public async Task GetByIdAsync_NotFound_ReturnsNull()
    {
        _repositoryMock
            .Setup(r => r.GetByIdAsync(99, default))
            .ReturnsAsync((Order?)null);

        var result = await _sut.GetByIdAsync(99);

        result.Should().BeNull();
    }
}
```

## `[TestCase]` — test parametrizzato

```csharp
[TestCase("")]
[TestCase("  ")]
[TestCase(null)]
public async Task CreateAsync_EmptyCustomer_ThrowsArgumentException(string? customer)
{
    var request = new CreateOrderRequest(customer!, 150.00m);
    var act = () => _sut.CreateAsync(request);
    await act.Should().ThrowAsync<ArgumentException>();
}

// TestCase con valori nominali — più leggibile
[TestCase(0,    TestName = "Total zero")]
[TestCase(-1,   TestName = "Total negativo")]
[TestCase(-100, TestName = "Total molto negativo")]
public async Task CreateAsync_InvalidTotal_ThrowsException(decimal total)
{
    var request = new CreateOrderRequest("Mario Rossi", total);
    var act = () => _sut.CreateAsync(request);
    await act.Should().ThrowAsync<ArgumentException>();
}
```

## `[TestCaseSource]` — dati da property o metodo

```csharp
// Equivalente di @MethodSource
private static IEnumerable<TestCaseData> InvalidRequestCases()
{
    yield return new TestCaseData(new CreateOrderRequest("", 100m))
        .SetName("Customer vuoto");
    yield return new TestCaseData(new CreateOrderRequest(null!, 100m))
        .SetName("Customer null");
    yield return new TestCaseData(new CreateOrderRequest("Mario", 0m))
        .SetName("Total zero");
}

[TestCaseSource(nameof(InvalidRequestCases))]
public async Task CreateAsync_InvalidRequest_ThrowsException(CreateOrderRequest request)
{
    var act = () => _sut.CreateAsync(request);
    await act.Should().ThrowAsync<ArgumentException>();
}
```

## `[OneTimeSetUp]` — setup una sola volta per fixture

```csharp
[TestFixture]
public class EfCoreRepositoryTests
{
    private AppDbContext _db;

    [OneTimeSetUp]   // ← equivalente di @BeforeAll
    public void OneTimeSetup()
    {
        var options = new DbContextOptionsBuilder<AppDbContext>()
            .UseInMemoryDatabase("NUnitTestDb")
            .Options;
        _db = new AppDbContext(options);
        _db.Database.EnsureCreated();
    }

    [OneTimeTearDown]  // ← equivalente di @AfterAll
    public void OneTimeTeardown() => _db.Dispose();

    [Test]
    public async Task AddAsync_ValidOrder_PersistsToDatabase()
    {
        var repo  = new EfCoreOrderRepository(_db);
        var order = new Order { Customer = "Test", Total = 50m, Status = OrderStatus.Pending };

        var result = await repo.AddAsync(order);

        result.Id.Should().BeGreaterThan(0);
    }
}
```

## Assert NUnit vs FluentAssertions

```csharp
// Assert NUnit — stile classico
Assert.That(result, Is.Not.Null);
Assert.That(result.Customer, Is.EqualTo("Mario Rossi"));
Assert.That(result.Total, Is.GreaterThan(0));
Assert.That(orders, Has.Count.EqualTo(2));
Assert.That(orders, Has.Some.Matches<Order>(o => o.Customer == "Mario Rossi"));

// FluentAssertions — stile fluente (consigliato)
result.Should().NotBeNull();
result.Customer.Should().Be("Mario Rossi");
result.Total.Should().BeGreaterThan(0);
orders.Should().HaveCount(2);
orders.Should().ContainSingle(o => o.Customer == "Mario Rossi");
```

## Spiegazione

**NUnit vs xUnit — isolamento** — xUnit crea una nuova istanza della classe per ogni test. NUnit crea una sola istanza e usa `[SetUp]`/`[TearDown]` per resettare lo stato. Entrambi gli approcci funzionano — NUnit richiede più attenzione allo stato condiviso tra test.

**`[TestCase]` vs `[InlineData]`** — `[TestCase]` di NUnit supporta `TestName` per dare un nome leggibile a ogni caso. `[InlineData]` di xUnit genera il nome automaticamente dai valori. Per casi complessi NUnit è più descrittivo; per casi semplici xUnit è più conciso.

---

## Confronto con Java

| JUnit 5 | NUnit |
|---|---|
| `@Test` | `[Test]` |
| `@ParameterizedTest` + `@ValueSource` | `[TestCase(value)]` |
| `@ParameterizedTest` + `@MethodSource` | `[TestCaseSource(nameof(method))]` |
| `@BeforeEach` | `[SetUp]` |
| `@AfterEach` | `[TearDown]` |
| `@BeforeAll` | `[OneTimeSetUp]` |
| `@AfterAll` | `[OneTimeTearDown]` |
| `@TestInstance(PER_CLASS)` | `[TestFixture]` default |
