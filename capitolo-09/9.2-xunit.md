# 9.2 xUnit in pratica: `[Fact]`, `[Theory]` e struttura dei test

> **Analogia Java**: equivalente di `@Test` e `@ParameterizedTest` in JUnit 5

---

## Introduzione

xUnit ha un'API minimale — pochi attributi, nessuna magia. Questo paragrafo mostra come scrivere unit test su `OrderService.Core` con xUnit e FluentAssertions, seguendo il pattern AAA (Arrange, Act, Assert).

---

## `[Fact]` — test semplice

```csharp
// Unit/Services/OrderServiceTests.cs
using FluentAssertions;
using Moq;
using OrderService.Core.Entities;
using OrderService.Core.Interfaces.Repositories;
using OrderService.Core.Services;
using Xunit;

namespace OrderService.Tests.Unit.Services;

public class OrderServiceTests
{
    private readonly Mock<IOrderRepository> _repositoryMock;
    private readonly OrderService           _sut;  // System Under Test

    // Il costruttore è l'equivalente di @BeforeEach
    public OrderServiceTests()
    {
        _repositoryMock = new Mock<IOrderRepository>();
        _sut            = new OrderService(_repositoryMock.Object, Mock.Of<ILogger<OrderService>>());
    }

    [Fact]
    public async Task CreateAsync_ValidRequest_ReturnsCreatedOrder()
    {
        // Arrange
        var request = new CreateOrderRequest("Mario Rossi", 150.00m);
        var expected = new Order { Id = 1, Customer = "Mario Rossi", Total = 150.00m };

        _repositoryMock
            .Setup(r => r.AddAsync(It.IsAny<Order>(), It.IsAny<CancellationToken>()))
            .ReturnsAsync(expected);

        // Act
        var result = await _sut.CreateAsync(request);

        // Assert
        result.Should().NotBeNull();
        result.Customer.Should().Be("Mario Rossi");
        result.Total.Should().Be(150.00m);
    }

    [Fact]
    public async Task CreateAsync_ValidRequest_CallsRepository()
    {
        // Arrange
        var request = new CreateOrderRequest("Mario Rossi", 150.00m);
        _repositoryMock
            .Setup(r => r.AddAsync(It.IsAny<Order>(), default))
            .ReturnsAsync(new Order());

        // Act
        await _sut.CreateAsync(request);

        // Assert — verifica che il repository sia stato chiamato una volta
        _repositoryMock.Verify(
            r => r.AddAsync(It.IsAny<Order>(), It.IsAny<CancellationToken>()),
            Times.Once);
    }

    [Fact]
    public async Task GetByIdAsync_NotFound_ReturnsNull()
    {
        // Arrange
        _repositoryMock
            .Setup(r => r.GetByIdAsync(99, default))
            .ReturnsAsync((Order?)null);

        // Act
        var result = await _sut.GetByIdAsync(99);

        // Assert
        result.Should().BeNull();
    }
}
```

## `[Theory]` — test parametrizzato

```csharp
[Theory]
[InlineData("")]
[InlineData("  ")]
[InlineData(null)]
public async Task CreateAsync_EmptyCustomer_ThrowsArgumentException(string? customer)
{
    // Arrange
    var request = new CreateOrderRequest(customer!, 150.00m);

    // Act
    var act = () => _sut.CreateAsync(request);

    // Assert
    await act.Should().ThrowAsync<ArgumentException>()
        .WithMessage("*Customer*");
}

[Theory]
[InlineData(0)]
[InlineData(-1)]
[InlineData(-999.99)]
public async Task CreateAsync_NegativeTotal_ThrowsArgumentException(decimal total)
{
    var request = new CreateOrderRequest("Mario Rossi", total);
    var act = () => _sut.CreateAsync(request);
    await act.Should().ThrowAsync<ArgumentException>();
}
```

## `[MemberData]` e `[ClassData]` — dati da collezioni

```csharp
// Per dati complessi — equivalente di @MethodSource in JUnit 5
public static IEnumerable<object[]> InvalidRequests =>
[
    [new CreateOrderRequest("", 100m),    "Customer vuoto"],
    [new CreateOrderRequest(null!, 100m), "Customer null"],
    [new CreateOrderRequest("Mario", 0m), "Total zero"],
    [new CreateOrderRequest("Mario", -1m),"Total negativo"],
];

[Theory]
[MemberData(nameof(InvalidRequests))]
public async Task CreateAsync_InvalidRequest_ThrowsException(
    CreateOrderRequest request, string scenario)
{
    var act = () => _sut.CreateAsync(request);
    await act.Should().ThrowAsync<ArgumentException>(
        because: $"il caso '{scenario}' dovrebbe fallire");
}
```

## Fixture condivisa — `IClassFixture<T>`

```csharp
// Fixtures/DatabaseFixture.cs
// Risorsa costosa condivisa tra tutti i test della classe
public class DatabaseFixture : IDisposable
{
    public AppDbContext Context { get; }

    public DatabaseFixture()
    {
        var options = new DbContextOptionsBuilder<AppDbContext>()
            .UseInMemoryDatabase(Guid.NewGuid().ToString())
            .Options;
        Context = new AppDbContext(options);
        Context.Database.EnsureCreated();
    }

    public void Dispose() => Context.Dispose();
}
```

```csharp
// Test con fixture condivisa — DbContext istanziato una sola volta
public class EfCoreOrderRepositoryTests : IClassFixture<DatabaseFixture>
{
    private readonly AppDbContext _db;

    public EfCoreOrderRepositoryTests(DatabaseFixture fixture)
    {
        _db = fixture.Context;
    }

    [Fact]
    public async Task AddAsync_ValidOrder_PersistsToDatabase()
    {
        var repo  = new EfCoreOrderRepository(_db);
        var order = new Order { Customer = "Mario Rossi", Total = 100m, Status = OrderStatus.Pending };

        var result = await repo.AddAsync(order);

        result.Id.Should().BeGreaterThan(0);
        _db.Orders.Should().ContainSingle(o => o.Id == result.Id);
    }
}
```

## FluentAssertions — asserzioni leggibili

```csharp
// Asserzioni su oggetti
result.Should().NotBeNull();
result.Should().BeOfType<Order>();
result.Customer.Should().Be("Mario Rossi");
result.Total.Should().BeGreaterThan(0);
result.Status.Should().Be(OrderStatus.Pending);

// Asserzioni su collezioni
orders.Should().HaveCount(2);
orders.Should().ContainSingle(o => o.Customer == "Mario Rossi");
orders.Should().BeInAscendingOrder(o => o.CreatedAt);
orders.Should().AllSatisfy(o => o.Total.Should().BePositive());

// Asserzioni su eccezioni
var act = () => sut.CreateAsync(invalidRequest);
await act.Should().ThrowAsync<ArgumentException>()
    .WithMessage("*non valido*");

// Asserzioni su Task
var task = sut.GetByIdAsync(1);
await task.Should().CompleteWithinAsync(TimeSpan.FromSeconds(1));
```

## Convenzione di naming

```
[Metodo]_[Scenario]_[Risultato atteso]

CreateAsync_ValidRequest_ReturnsCreatedOrder
CreateAsync_EmptyCustomer_ThrowsArgumentException
GetByIdAsync_NotFound_ReturnsNull
GetByIdAsync_ExistingId_ReturnsOrder
```

È la convenzione più diffusa nell'ecosistema .NET — rende il nome del test auto-documentante.

---

## Spiegazione

**Costruttore vs `[SetUp]`** — xUnit istanzia una nuova classe per ogni test. Il costruttore è quindi il luogo naturale per l'inizializzazione — equivalente a `@BeforeEach` senza annotazioni. `IDisposable.Dispose` è il `@AfterEach`.

**`Mock.Of<T>()` vs `new Mock<T>()`** — `Mock.Of<T>()` crea un mock che non verifica le chiamate — utile per dipendenze che non ti interessano nel test corrente (es. `ILogger`). `new Mock<T>()` crea un mock su cui puoi fare `Setup` e `Verify`.

**`It.IsAny<T>()` vs valore specifico** — usa `It.IsAny<T>()` per dipendenze che non fanno parte dello scenario che stai testando. Usa valori specifici quando il test riguarda proprio quel valore — rende il test più preciso.

## Confronto con Java

| JUnit 5 | xUnit |
|---|---|
| `@Test` | `[Fact]` |
| `@ParameterizedTest` + `@ValueSource` | `[Theory]` + `[InlineData]` |
| `@ParameterizedTest` + `@MethodSource` | `[Theory]` + `[MemberData]` |
| `@BeforeEach` | Costruttore della classe |
| `@AfterEach` | `IDisposable.Dispose` |
| `@BeforeAll` / `@AfterAll` | `IClassFixture<T>` |
| `assertThat(x).isEqualTo(y)` (AssertJ) | `x.Should().Be(y)` (FluentAssertions) |
| `assertThrows` | `.Should().ThrowAsync<T>()` |
| `System.out.println` nei test | `ITestOutputHelper.WriteLine` |
