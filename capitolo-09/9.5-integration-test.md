# 9.5 Integration test con `WebApplicationFactory`

> **Analogia Java**: equivalente di `@SpringBootTest`

---

## Introduzione

I test unitari testano la logica in isolamento. I test di integrazione testano l'intera stack — controller, middleware, DI, serializzazione JSON — senza un server HTTP reale. In Java era `@SpringBootTest` con `MockMvc` o `TestRestTemplate`. In .NET è `WebApplicationFactory<TProgram>` — avvia l'applicazione in memoria e ti dà un `HttpClient` per fare chiamate reali agli endpoint.

---

## Come si faceva in Java

```java
@SpringBootTest(webEnvironment = SpringBootTest.WebEnvironment.RANDOM_PORT)
@AutoConfigureMockMvc
class OrderControllerTest {

    @Autowired
    private MockMvc mockMvc;

    @MockBean
    private OrderService orderService;

    @Test
    void shouldReturnOrders() throws Exception {
        when(orderService.findAll()).thenReturn(List.of(new Order(1L, "Mario", 100.0)));

        mockMvc.perform(get("/orders")
                .header("Authorization", "Bearer " + token))
            .andExpect(status().isOk())
            .andExpect(jsonPath("$[0].customer").value("Mario"));
    }
}
```

## `WebApplicationFactory` base

```csharp
// Integration/OrderControllerTests.cs
using System.Net;
using System.Net.Http.Json;
using FluentAssertions;
using Microsoft.AspNetCore.Mvc.Testing;
using Xunit;

namespace OrderService.Tests.Integration;

public class OrderControllerTests : IClassFixture<WebApplicationFactory<Program>>
{
    private readonly HttpClient _client;

    public OrderControllerTests(WebApplicationFactory<Program> factory)
    {
        _client = factory.CreateClient();
    }

    [Fact]
    public async Task GetAll_Unauthenticated_Returns401()
    {
        var response = await _client.GetAsync("/orders");
        response.StatusCode.Should().Be(HttpStatusCode.Unauthorized);
    }

    [Fact]
    public async Task GetAll_Authenticated_Returns200()
    {
        // Aggiunge il token JWT all'header
        _client.DefaultRequestHeaders.Authorization =
            new System.Net.Http.Headers.AuthenticationHeaderValue("Bearer", TestTokenHelper.Generate());

        var response = await _client.GetAsync("/orders");
        response.StatusCode.Should().Be(HttpStatusCode.OK);
    }
}
```

## `WebApplicationFactory` con override dei servizi

Per i test di integrazione vuoi sostituire il database reale con uno in-memory — o mockare dipendenze esterne:

```csharp
// Fixtures/TestWebApplicationFactory.cs
using Microsoft.AspNetCore.Hosting;
using Microsoft.AspNetCore.Mvc.Testing;
using Microsoft.EntityFrameworkCore;
using Microsoft.Extensions.DependencyInjection;
using OrderService.Infrastructure.Persistence;

namespace OrderService.Tests.Fixtures;

public class TestWebApplicationFactory : WebApplicationFactory<Program>
{
    protected override void ConfigureWebHost(IWebHostBuilder builder)
    {
        builder.ConfigureServices(services =>
        {
            // Rimuovi il DbContext reale
            var descriptor = services.SingleOrDefault(
                d => d.ServiceType == typeof(DbContextOptions<AppDbContext>));
            if (descriptor is not null)
                services.Remove(descriptor);

            // Aggiungi un DbContext in-memory per i test
            services.AddDbContext<AppDbContext>(options =>
                options.UseInMemoryDatabase($"TestDb-{Guid.NewGuid()}"));

            // Sostituisci IInventoryClient con un mock — rimuovi prima il typed client reale
            var inventoryDescriptor = services.SingleOrDefault(
                d => d.ServiceType == typeof(IInventoryClient));
            if (inventoryDescriptor is not null) services.Remove(inventoryDescriptor);

            var mockInventory = new Mock<IInventoryClient>();
            mockInventory
                .Setup(c => c.IsAvailableAsync(It.IsAny<int>(), It.IsAny<int>(), It.IsAny<CancellationToken>()))
                .ReturnsAsync(true);
            services.AddScoped<IInventoryClient>(_ => mockInventory.Object);

            // Applica le migration in-memory
            var sp = services.BuildServiceProvider();
            using var scope = sp.CreateScope();
            var db = scope.ServiceProvider.GetRequiredService<AppDbContext>();
            db.Database.EnsureCreated();
        });

        builder.UseEnvironment("Testing");
    }
}
```

```csharp
// Integration/OrderControllerTests.cs — usa la factory custom
public class OrderControllerTests : IClassFixture<TestWebApplicationFactory>
{
    private readonly HttpClient _client;

    public OrderControllerTests(TestWebApplicationFactory factory)
    {
        _client = factory.CreateClient();
    }

    [Fact]
    public async Task CreateOrder_ValidRequest_Returns201()
    {
        _client.DefaultRequestHeaders.Authorization =
            new AuthenticationHeaderValue("Bearer", TestTokenHelper.Generate("Admin"));

        var request = new { customer = "Mario Rossi", total = 150.00 };
        var response = await _client.PostAsJsonAsync("/orders", request);

        response.StatusCode.Should().Be(HttpStatusCode.Created);

        var order = await response.Content.ReadFromJsonAsync<OrderResponse>();
        order.Should().NotBeNull();
        order!.Customer.Should().Be("Mario Rossi");
        order.Id.Should().BeGreaterThan(0);
    }

    [Fact]
    public async Task CreateOrder_MissingCustomer_Returns400()
    {
        _client.DefaultRequestHeaders.Authorization =
            new AuthenticationHeaderValue("Bearer", TestTokenHelper.Generate("Admin"));

        var request = new { total = 150.00 };  // customer mancante
        var response = await _client.PostAsJsonAsync("/orders", request);

        response.StatusCode.Should().Be(HttpStatusCode.BadRequest);

        var problem = await response.Content.ReadFromJsonAsync<ProblemDetails>();
        problem!.Status.Should().Be(400);
    }

    [Fact]
    public async Task ConfirmOrder_NonAdmin_Returns403()
    {
        // Crea un ordine come Admin
        _client.DefaultRequestHeaders.Authorization =
            new AuthenticationHeaderValue("Bearer", TestTokenHelper.Generate("Admin"));
        var createResponse = await _client.PostAsJsonAsync("/orders",
            new { customer = "Mario Rossi", total = 150.00 });
        var created = await createResponse.Content.ReadFromJsonAsync<OrderResponse>();

        // Prova a confermarlo come User (ruolo non autorizzato)
        _client.DefaultRequestHeaders.Authorization =
            new AuthenticationHeaderValue("Bearer", TestTokenHelper.Generate("User"));

        var response = await _client.PostAsync($"/orders/{created!.Id}/confirm", null);

        response.StatusCode.Should().Be(HttpStatusCode.Forbidden);
    }

}
```

> 💡 **Nota**: il test `ConfirmOrder_NonAdmin_Returns403` usa `POST /orders/{id}/confirm`, l'endpoint definito in 9.7, invece di `DELETE /orders/{id}` che non è mai stato introdotto nel corso. Questo garantisce che il test eserciti un endpoint realmente esistente.

## `TestTokenHelper` — classe helper condivisa

`GenerateTestToken` diventa una classe statica riutilizzabile dai test di integrazione in tutti i capitoli successivi:

```csharp
// Tests/Helpers/TestTokenHelper.cs
using System.IdentityModel.Tokens.Jwt;
using System.Security.Claims;
using System.Text;
using Microsoft.IdentityModel.Tokens;

namespace OrderService.Tests.Helpers;

public static class TestTokenHelper
{
    public static string Generate(string role = "User")
    {
        var key         = new SymmetricSecurityKey(Encoding.UTF8.GetBytes("test-key-almeno-32-byte-per-hs256!!"));
        var credentials = new SigningCredentials(key, SecurityAlgorithms.HmacSha256);
        var token       = new JwtSecurityToken(
            issuer:             "order-service",
            audience:           "order-service-api",
            claims:             [new Claim(ClaimTypes.Role, role), new Claim("sub", "test-user-1")],
            expires:            DateTime.UtcNow.AddHours(1),
            signingCredentials: credentials);
        return new JwtSecurityTokenHandler().WriteToken(token);
    }
}
```

## `appsettings.Testing.json`

```json
// Api/appsettings.Testing.json
{
  "Jwt": {
    "Key": "test-key-almeno-32-byte-per-hs256!!",
    "Issuer": "order-service",
    "Audience": "order-service-api"
  },
  "Serilog": {
    "MinimumLevel": {
      "Default": "Warning"
    }
  }
}
```

Il file deve essere copiato nella directory di output del progetto `Api` — altrimenti `UseEnvironment("Testing")` non lo trova e i test autenticati falliscono con 401:

```xml
<!-- OrderService.Api.csproj -->
<ItemGroup>
  <Content Include="appsettings.Testing.json" CopyToOutputDirectory="PreserveNewest" />
</ItemGroup>
```

## Spiegazione

**`IClassFixture<TestWebApplicationFactory>`** — la factory viene creata una sola volta e condivisa tra tutti i test della classe. Questo è importante: avviare l'applicazione è costoso — non vuoi farlo per ogni test. Il database in-memory è un `Guid` univoco per evitare interferenze tra test paralleli.

**`UseInMemoryDatabase($"TestDb-{Guid.NewGuid()}")`** — ogni istanza della factory ha il proprio database in-memory. Se hai più classi di test che usano la stessa factory, il database è condiviso — fai attenzione all'ordine di esecuzione. Con un `Guid` unico ogni factory parte con un database pulito.

**`builder.UseEnvironment("Testing")`** — carica `appsettings.Testing.json` invece di `appsettings.Development.json`. Permette di usare chiavi JWT di test e sopprimere il logging verboso.

**Integration test vs unit test** — i test di integrazione sono più lenti ma testano il comportamento reale: routing, middleware, serializzazione JSON, validazione FluentValidation, gestione errori. Un test che verifica che un `POST /orders` con body invalido restituisca un Problem Details RFC 7807 non è testabile con i soli unit test.

## Confronto con Java

| Spring Boot Test | .NET WebApplicationFactory |
|---|---|
| `@SpringBootTest` | `WebApplicationFactory<Program>` |
| `MockMvc` | `HttpClient` da `factory.CreateClient()` |
| `@MockBean` | `services.Remove(descriptor)` + re-registrazione |
| `@ActiveProfiles("test")` | `builder.UseEnvironment("Testing")` |
| `mockMvc.perform(get("/orders"))` | `_client.GetAsync("/orders")` |
| `.andExpect(status().isOk())` | `response.StatusCode.Should().Be(HttpStatusCode.OK)` |
| `.andExpect(jsonPath("$.customer").value("x"))` | `ReadFromJsonAsync<T>()` + FluentAssertions |
| `H2` in-memory per i test | `UseInMemoryDatabase` o Testcontainers (9.6) |
