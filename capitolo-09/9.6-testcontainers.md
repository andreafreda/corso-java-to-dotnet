# 9.6 Test del database con Testcontainers

> **Analogia Java**: equivalente di Testcontainers for Java

---

## Introduzione

`UseInMemoryDatabase` nei test di integrazione è veloce ma impreciso — non testa query complesse, constraints, migrations, tipi specifici di SQL Server. **Testcontainers** avvia un container Docker reale durante i test e lo distrugge alla fine. In Java era già disponibile — in .NET è supportato ufficialmente dal team Microsoft da .NET 8.

---

## Come si faceva in Java

```java
@Testcontainers
@SpringBootTest
class OrderRepositoryTest {

    @Container
    static PostgreSQLContainer<?> postgres = new PostgreSQLContainer<>("postgres:16")
        .withDatabaseName("testdb")
        .withUsername("user")
        .withPassword("secret");

    @DynamicPropertySource
    static void configureProperties(DynamicPropertyRegistry registry) {
        registry.add("spring.datasource.url", postgres::getJdbcUrl);
        registry.add("spring.datasource.username", postgres::getUsername);
        registry.add("spring.datasource.password", postgres::getPassword);
    }

    @Test
    void shouldPersistOrder() {
        // Test su PostgreSQL reale
    }
}
```

## Fixture con Testcontainers

```csharp
// Fixtures/SqlServerFixture.cs
using DotNet.Testcontainers.Builders;
using Microsoft.EntityFrameworkCore;
using OrderService.Infrastructure.Persistence;
using Testcontainers.MsSql;
using Xunit;

namespace OrderService.Tests.Fixtures;

// IAsyncLifetime per setup/teardown asincrono
public class SqlServerFixture : IAsyncLifetime
{
    private readonly MsSqlContainer _container = new MsSqlBuilder()
        .WithImage("mcr.microsoft.com/azure-sql-edge:latest")
        .WithPassword("TestPassword123!")
        .Build();

    public AppDbContext Context { get; private set; } = null!;

    public async Task InitializeAsync()
    {
        await _container.StartAsync();

        var options = new DbContextOptionsBuilder<AppDbContext>()
            .UseSqlServer(_container.GetConnectionString())
            .Options;

        Context = new AppDbContext(options);
        await Context.Database.MigrateAsync();  // applica le migration reali
    }

    public async Task DisposeAsync()
    {
        await Context.DisposeAsync();
        await _container.DisposeAsync();
    }
}
```

## Test con database reale

```csharp
// Integration/Repositories/EfCoreOrderRepositoryTests.cs
using FluentAssertions;
using OrderService.Core.Entities;
using OrderService.Infrastructure.Repositories;
using OrderService.Tests.Fixtures;
using Xunit;

namespace OrderService.Tests.Integration.Repositories;

public class EfCoreOrderRepositoryTests : IClassFixture<SqlServerFixture>
{
    private readonly AppDbContext _db;

    public EfCoreOrderRepositoryTests(SqlServerFixture fixture)
    {
        _db = fixture.Context;
        // Pulisci i dati prima di ogni test
        _db.Orders.RemoveRange(_db.Orders);
        _db.SaveChanges();
    }

    [Fact]
    public async Task AddAsync_ValidOrder_AssignsId()
    {
        var repo  = new EfCoreOrderRepository(_db);
        var order = new Order
        {
            Customer  = "Mario Rossi",
            Total     = 150.00m,
            Status    = OrderStatus.Pending,
            CreatedAt = DateTime.UtcNow
        };

        var result = await repo.AddAsync(order);

        result.Id.Should().BeGreaterThan(0);
    }

    [Fact]
    public async Task GetByIdAsync_ExistingOrder_ReturnsOrder()
    {
        // Arrange — inserisci direttamente nel DB
        var order = new Order { Customer = "Test", Total = 50m, Status = OrderStatus.Pending };
        _db.Orders.Add(order);
        await _db.SaveChangesAsync();

        var repo = new EfCoreOrderRepository(_db);

        // Act
        var result = await repo.GetByIdAsync(order.Id);

        // Assert
        result.Should().NotBeNull();
        result!.Customer.Should().Be("Test");
    }

    [Fact]
    public async Task GetAllAsync_MultipleOrders_ReturnsAll()
    {
        _db.Orders.AddRange(
            new Order { Customer = "Mario",  Total = 100m, Status = OrderStatus.Pending },
            new Order { Customer = "Giulia", Total = 200m, Status = OrderStatus.Confirmed });
        await _db.SaveChangesAsync();

        var repo   = new EfCoreOrderRepository(_db);
        var orders = await repo.GetAllAsync();

        orders.Should().HaveCount(2);
    }
}
```

## `WebApplicationFactory` con Testcontainers

Puoi combinare `WebApplicationFactory` e Testcontainers per test end-to-end con database reale:

```csharp
// Fixtures/IntegrationTestFactory.cs
public class IntegrationTestFactory : WebApplicationFactory<Program>, IAsyncLifetime
{
    private readonly MsSqlContainer _container = new MsSqlBuilder()
        .WithImage("mcr.microsoft.com/azure-sql-edge:latest")
        .WithPassword("TestPassword123!")
        .Build();

    public async Task InitializeAsync()
    {
        await _container.StartAsync();
    }

    protected override void ConfigureWebHost(IWebHostBuilder builder)
    {
        builder.ConfigureServices(services =>
        {
            var descriptor = services.SingleOrDefault(
                d => d.ServiceType == typeof(DbContextOptions<AppDbContext>));
            if (descriptor is not null) services.Remove(descriptor);

            // Usa SQL Server reale dal container
            services.AddDbContext<AppDbContext>(options =>
                options.UseSqlServer(_container.GetConnectionString()));

            // Applica migrations
            var sp = services.BuildServiceProvider();
            using var scope = sp.CreateScope();
            var db = scope.ServiceProvider.GetRequiredService<AppDbContext>();
            db.Database.Migrate();
        });

        builder.UseEnvironment("Testing");
    }

    public async Task DisposeAsync()
    {
        await _container.DisposeAsync();
        Dispose();  // WebApplicationFactory implementa IDisposable, non IAsyncDisposable
    }
}
```

```csharp
// Integration/OrderApiTests.cs — test end-to-end con SQL Server reale
public class OrderApiTests : IClassFixture<IntegrationTestFactory>
{
    private readonly HttpClient _client;

    public OrderApiTests(IntegrationTestFactory factory)
    {
        _client = factory.CreateClient();
        _client.DefaultRequestHeaders.Authorization =
            new AuthenticationHeaderValue("Bearer", TestTokenHelper.Generate("Admin"));
    }

    [Fact]
    public async Task CreateAndRetrieveOrder_EndToEnd()
    {
        // Create
        var createResponse = await _client.PostAsJsonAsync("/orders",
            new { customer = "Mario Rossi", total = 150.00 });
        createResponse.StatusCode.Should().Be(HttpStatusCode.Created);

        var created = await createResponse.Content.ReadFromJsonAsync<OrderResponse>();

        // Retrieve
        var getResponse = await _client.GetAsync($"/orders/{created!.Id}");
        getResponse.StatusCode.Should().Be(HttpStatusCode.OK);

        var retrieved = await getResponse.Content.ReadFromJsonAsync<OrderResponse>();
        retrieved!.Customer.Should().Be("Mario Rossi");
        retrieved.Total.Should().Be(150.00m);
    }
}
```

## Spiegazione

**`IAsyncLifetime`** — xUnit non ha `[OneTimeSetUp]` asincrono. `IAsyncLifetime` è l'alternativa: `InitializeAsync` viene chiamato prima del primo test, `DisposeAsync` dopo l'ultimo. Testcontainers richiede async — `IAsyncLifetime` è il pattern corretto.

**Container condiviso tra test** — `IClassFixture<SqlServerFixture>` crea il container una sola volta per classe. Se hai molte classi di test, valuta `ICollectionFixture<T>` — condivide il container tra più classi, riducendo drasticamente il tempo di startup.

**Migrations reali** — `db.Database.Migrate()` applica le stesse migration che girano in produzione. È la differenza chiave rispetto a `UseInMemoryDatabase` — testi constraint, indici, tipi di colonna SQL Server reali.

**Costo temporale** — avviare un container Docker richiede 5-15 secondi. Usalo per i test di integrazione più importanti — non per ogni test unitario. Un buon mix: unit test veloci per la logica business, Testcontainers per repository e API end-to-end.

## Confronto con Java

| Testcontainers for Java | Testcontainers for .NET |
|---|---|
| `@Testcontainers` + `@Container` | `IAsyncLifetime` + `MsSqlContainer` |
| `PostgreSQLContainer` / `MSSQLServerContainer` | `PostgreSqlContainer` / `MsSqlContainer` |
| `@DynamicPropertySource` per connection string | `ConfigureWebHost` + `services.Remove(descriptor)` |
| `container.getJdbcUrl()` | `_container.GetConnectionString()` |
| Avvio automatico con `@Testcontainers` | `await _container.StartAsync()` in `InitializeAsync` |
| `@BeforeAll` / `@AfterAll` | `IAsyncLifetime.InitializeAsync` / `DisposeAsync` |
| `ICollectionFixture<T>` (xUnit) | `ICollectionFixture<T>` — stesso pattern |
