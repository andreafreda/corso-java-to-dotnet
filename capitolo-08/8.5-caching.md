# 8.5 Caching in .NET: IMemoryCache, IDistributedCache e Redis

> **Analogia Java**: equivalente di Spring Cache + Redis

---

## Introduzione

`OrderService` interroga `InventoryService` per ogni ordine. Se la disponibilità non cambia ogni millisecondo, ha senso cachare la risposta. .NET offre due livelli di cache: **`IMemoryCache`** (in-process, singola istanza) e **`IDistributedCache`** (condivisa tra più istanze, tipicamente Redis). In Java era Spring Cache con `@Cacheable`.

---

## Come si faceva in Java

```java
// Spring Cache — @Cacheable sul metodo
@Service
public class InventoryService {

    @Cacheable(value = "inventory", key = "#productId")
    public boolean isAvailable(int productId, int quantity) {
        return inventoryClient.checkAvailability(productId, quantity);
    }

    @CacheEvict(value = "inventory", key = "#productId")
    public void invalidate(int productId) { }
}
```

```yaml
# application.yml — Redis come cache store
spring:
  cache:
    type: redis
  data:
    redis:
      host: localhost
      port: 6379
```

## `IMemoryCache` — cache in-process

Adatta per dati che non cambiano spesso e che non devono essere condivisi tra istanze:

```csharp
// Api/Program.cs
builder.Services.AddMemoryCache();
```

```csharp
// Infrastructure/Http/InventoryClient.cs
using Microsoft.Extensions.Caching.Memory;
using OrderService.Core.Interfaces;

public class InventoryClient : IInventoryClient
{
    private readonly HttpClient               _client;
    private readonly IMemoryCache             _cache;
    private readonly ILogger<InventoryClient> _logger;

    public InventoryClient(HttpClient client, IMemoryCache cache, ILogger<InventoryClient> logger)
    {
        _client = client;
        _cache  = cache;
        _logger = logger;
    }

    public async Task<bool> IsAvailableAsync(int productId, int quantity, CancellationToken ct = default)
    {
        var cacheKey = $"inventory:{productId}";

        if (_cache.TryGetValue(cacheKey, out bool cached))
        {
            _logger.LogDebug("Cache hit per productId={ProductId}", productId);
            return cached;
        }

        var response = await _client.GetAsync(
            $"/inventory/{productId}/available?quantity={quantity}", ct);
        response.EnsureSuccessStatusCode();
        var available = await response.Content.ReadFromJsonAsync<bool>(ct);

        _cache.Set(cacheKey, available, new MemoryCacheEntryOptions
        {
            AbsoluteExpirationRelativeToNow = TimeSpan.FromMinutes(5),
            SlidingExpiration               = TimeSpan.FromMinutes(2)
        });

        return available;
    }
}
```

## `IDistributedCache` con Redis — cache condivisa

Per deployment con più istanze la `IMemoryCache` non funziona — ogni istanza ha la propria cache. Redis condivide la cache tra tutte le istanze:

```bash
dotnet add OrderService.Infrastructure package Microsoft.Extensions.Caching.StackExchangeRedis
```

```yaml
# docker-compose.yml
services:
  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"
```

```csharp
// Api/Program.cs
builder.Services.AddStackExchangeRedisCache(options =>
{
    options.Configuration = builder.Configuration.GetConnectionString("Redis");
    options.InstanceName  = "OrderService:";
});
```

```json
// appsettings.Development.json
{
  "ConnectionStrings": {
    "Redis": "localhost:6379"
  }
}
```

```csharp
// Infrastructure/Http/InventoryClient.cs — con IDistributedCache
using Microsoft.Extensions.Caching.Distributed;
using System.Text.Json;
using OrderService.Core.Interfaces;

public class InventoryClient : IInventoryClient
{
    private readonly HttpClient               _client;
    private readonly IDistributedCache        _cache;
    private readonly ILogger<InventoryClient> _logger;

    public InventoryClient(HttpClient client, IDistributedCache cache, ILogger<InventoryClient> logger)
    {
        _client = client;
        _cache  = cache;
        _logger = logger;
    }

    public async Task<bool> IsAvailableAsync(int productId, int quantity, CancellationToken ct = default)
    {
        var cacheKey = $"inventory:{productId}";

        var cached = await _cache.GetStringAsync(cacheKey, ct);
        if (cached is not null)
        {
            _logger.LogDebug("Cache hit per productId={ProductId}", productId);
            return JsonSerializer.Deserialize<bool>(cached);
        }

        var response = await _client.GetAsync(
            $"/inventory/{productId}/available?quantity={quantity}", ct);
        response.EnsureSuccessStatusCode();
        var available = await response.Content.ReadFromJsonAsync<bool>(ct);

        await _cache.SetStringAsync(cacheKey,
            JsonSerializer.Serialize(available),
            new DistributedCacheEntryOptions
            {
                AbsoluteExpirationRelativeToNow = TimeSpan.FromMinutes(5)
            }, ct);

        return available;
    }

    public async Task InvalidateAsync(int productId, CancellationToken ct = default)
    {
        await _cache.RemoveAsync($"inventory:{productId}", ct);
    }
}
```

## Cache-aside pattern — caching trasparente nel repository

```csharp
// Infrastructure/Repositories/CachedOrderRepository.cs
// Decorator che aggiunge cache a IOrderRepository senza modificarlo
public class CachedOrderRepository : IOrderRepository
{
    private readonly IOrderRepository  _inner;
    private readonly IDistributedCache _cache;

    public CachedOrderRepository(IOrderRepository inner, IDistributedCache cache)
    {
        _inner = inner;
        _cache = cache;
    }

    public async Task<Order?> GetByIdAsync(int id, CancellationToken ct = default)
    {
        var cacheKey = $"order:{id}";
        var cached   = await _cache.GetStringAsync(cacheKey, ct);

        if (cached is not null)
            return JsonSerializer.Deserialize<Order>(cached);

        var order = await _inner.GetByIdAsync(id, ct);

        if (order is not null)
            await _cache.SetStringAsync(cacheKey,
                JsonSerializer.Serialize(order),
                new DistributedCacheEntryOptions
                {
                    AbsoluteExpirationRelativeToNow = TimeSpan.FromMinutes(10)
                }, ct);

        return order;
    }

    public Task<IEnumerable<Order>> GetAllAsync(CancellationToken ct = default)
        => _inner.GetAllAsync(ct);  // lista non cachata — cambia troppo spesso

    public Task<Order> AddAsync(Order order, CancellationToken ct = default)
        => _inner.AddAsync(order, ct);  // ordine appena creato: nessuna cache da invalidare

    public async Task DeleteAsync(int id, CancellationToken ct = default)
    {
        await _inner.DeleteAsync(id, ct);
        await _cache.RemoveAsync($"order:{id}", ct);  // invalida l'entry del singolo ordine
    }

    public Task UpdateAsync(Order order, CancellationToken ct = default)
        => _inner.UpdateAsync(order, ct);

    public Task<Order> AddWithOutboxAsync(Order order, OrderCreatedEvent eventData, CancellationToken ct = default)
        => _inner.AddWithOutboxAsync(order, eventData, ct);
}
```

```csharp
// Infrastructure/DependencyInjection.cs — decorator pattern
services.AddScoped<EfCoreOrderRepository>();
services.AddScoped<IOrderRepository>(sp =>
    new CachedOrderRepository(
        sp.GetRequiredService<EfCoreOrderRepository>(),
        sp.GetRequiredService<IDistributedCache>()));
```

## Spiegazione

**`IMemoryCache` vs `IDistributedCache`** — se hai una sola istanza del servizio, `IMemoryCache` è più semplice e veloce (nessuna serializzazione, nessun round trip di rete). Con più istanze (Kubernetes, scaling orizzontale) devi usare `IDistributedCache` — altrimenti le istanze hanno cache diverse e i dati diventano inconsistenti.

**`AbsoluteExpiration` vs `SlidingExpiration`** — assoluta: il dato scade dopo N minuti dall'inserimento. Sliding: il dato scade dopo N minuti dall'ultimo accesso — viene rinnovata ad ogni lettura. Per dati di inventory usa l'assoluta — non vuoi che i dati stantii restino per sempre solo perché vengono letti frequentemente.

**Serializzazione esplicita** — `IDistributedCache` lavora con `byte[]` o `string`. A differenza di Spring Cache, non c'è serializzazione automatica — devi serializzare/deserializzare manualmente con `JsonSerializer`. È esplicito ma ti dà controllo totale sul formato.

**Cache-aside vs `@Cacheable`** — Spring Cache con `@Cacheable` è trasparente: Hibernate/Spring gestiscono tutto. .NET non ha l'equivalente dichiarativo — il pattern cache-aside (controlla, leggi, scrivi) è esplicito. Il decorator pattern su `IOrderRepository` ottiene lo stesso risultato con codice pulito.

## Confronto con Java

| Spring Cache + Redis | .NET IDistributedCache + Redis |
|---|---|
| `@Cacheable(value="x", key="#id")` | Cache-aside esplicito nel metodo |
| `@CacheEvict` | `_cache.RemoveAsync(key)` |
| `spring.cache.type=redis` | `AddStackExchangeRedisCache(options => ...)` |
| `RedisTemplate` / `StringRedisTemplate` | `IDistributedCache.GetStringAsync/SetStringAsync` |
| Serializzazione automatica con `Jackson` | Serializzazione manuale con `JsonSerializer` |
| `spring.data.redis.host` in `application.yml` | `ConnectionStrings:Redis` in `appsettings.json` |
