# 8.7 Feature Flags con Microsoft.FeatureManagement

> **Analogia Java**: equivalente di Togglz / FF4J in Spring

---

## Introduzione

Vuoi rilasciare la validazione dell'inventory in produzione solo per il 10% degli utenti, o disabilitarla rapidamente se causa problemi — senza un deploy. I **feature flags** risolvono questo: un booleano configurabile a runtime che abilita o disabilita un blocco di codice. In Java si usava Togglz o FF4J. In .NET c'è **Microsoft.FeatureManagement** — integrato con `IConfiguration` e Azure App Configuration.

---

## Come si faceva in Java

```java
// Togglz — definizione feature
public enum Features implements Feature {
    @EnabledByDefault
    INVENTORY_CHECK,
    NEW_PRICING_ENGINE;

    public boolean isActive() {
        return FeatureContext.getFeatureManager().isActive(this);
    }
}
```

```java
// Uso nel codice
@Service
public class OrderService {

    public Order createOrder(CreateOrderRequest request) {
        if (Features.INVENTORY_CHECK.isActive()) {
            boolean available = inventoryClient.isAvailable(request.productId(), request.quantity());
            if (!available) throw new InsufficientStockException();
        }
        return orderRepository.save(new Order(request));
    }
}
```

## Definizione delle feature flags

```json
// appsettings.json
{
  "FeatureManagement": {
    "InventoryCheck":    true,
    "NewPricingEngine":  false,
    "BetaUserExperience": {
      "EnabledFor": [
        {
          "Name": "Percentage",
          "Parameters": { "Value": 10 }
        }
      ]
    }
  }
}
```

```csharp
// Core/FeatureFlags.cs — costanti per evitare stringhe magiche
namespace OrderService.Core;

public static class FeatureFlags
{
    public const string InventoryCheck     = "InventoryCheck";
    public const string NewPricingEngine   = "NewPricingEngine";
    public const string BetaUserExperience = "BetaUserExperience";
}
```

## Uso nel service

```csharp
// Core/Services/OrderService.cs
using Microsoft.FeatureManagement;

public class OrderService : IOrderService
{
    private readonly IOrderRepository     _repository;
    private readonly IInventoryClient     _inventoryClient;  // interfaccia — non implementazione
    private readonly IFeatureManager      _features;
    private readonly ILogger<OrderService> _logger;

    public async Task<Order> CreateAsync(CreateOrderRequest request, CancellationToken ct)
    {
        // Feature flag — controlla inventory solo se abilitato
        if (await _features.IsEnabledAsync(FeatureFlags.InventoryCheck))
        {
            var available = await _inventoryClient.IsAvailableAsync(
                request.ProductId, request.Quantity, ct);

            if (!available)
                throw new InvalidOperationException(
                    $"Prodotto {request.ProductId} non disponibile");
        }
        else
        {
            _logger.LogWarning("InventoryCheck disabilitato — skip verifica disponibilità");
        }

        var order = new Order
        {
            Customer  = request.Customer,
            Total     = request.Total,
            Status    = OrderStatus.Pending,
            CreatedAt = DateTime.UtcNow
        };

        return await _repository.AddAsync(order, ct);
    }
}
```

## Uso sugli endpoint — `[FeatureGate]`

```csharp
// Disabilita l'intero endpoint se la feature è off — restituisce 404
[HttpPost("beta/bulk-order")]
[FeatureGate(FeatureFlags.NewPricingEngine)]
public async Task<IActionResult> CreateBulkOrder(/* ... */)
{
    // Questo endpoint è raggiungibile solo se NewPricingEngine è true
}
```

## Feature flag contestuale — per utente

```csharp
// Uso con contesto utente — la feature è abilitata per percentuale di request
public async Task<IActionResult> GetDashboard()
{
    if (await _features.IsEnabledAsync(FeatureFlags.BetaUserExperience))
    {
        return Ok(await _orderService.GetBetaDashboardAsync());
    }

    return Ok(await _orderService.GetDashboardAsync());
}
```

> Il filtro `Percentage` è stateless — non usa l'identità dell'utente. Per rollout stabili per utente specifico usa il filtro `Targeting` di Azure App Configuration, che richiede un `ITargetingContextAccessor` con lo `UserId`.

## Feature flags con Azure App Configuration

In produzione le feature flags si gestiscono da Azure App Configuration — nessun deploy per cambiare un valore:

```csharp
// Api/Program.cs — Azure App Configuration con feature flags
builder.Configuration.AddAzureAppConfiguration(options =>
{
    options.Connect(builder.Configuration["AppConfiguration:ConnectionString"])
           .UseFeatureFlags(flagOptions =>
           {
               flagOptions.SetCacheExpiry(TimeSpan.FromSeconds(30));
           });
});

builder.Services.AddAzureAppConfiguration();
builder.Services.AddFeatureManagement();
```

Dal portale Azure App Configuration puoi attivare o disattivare `InventoryCheck` in tempo reale — `OrderService` lo rileva entro 30 secondi senza restart.

---

## Spiegazione

**`IFeatureManager` vs `IConfiguration`** — potresti leggere `"FeatureManagement:InventoryCheck"` direttamente da `IConfiguration`. `IFeatureManager` aggiunge filtri condizionali (percentage rollout, utenti specifici, orari) che `IConfiguration` non supporta. Usa sempre `IFeatureManager`.

**`[FeatureGate]` restituisce 404** — quando una feature è disabilitata, `[FeatureGate]` non restituisce 403 ma 404 — l'endpoint "non esiste". È una scelta deliberata: non vuoi rivelare che esiste una funzionalità non ancora rilasciata.

**Percentage rollout** — il filtro `Percentage` è stateless — lo stesso utente potrebbe ottenere risultati diversi tra una request e l'altra. Per rollout stabili per utente usa il filtro `Targeting` (disponibile con Azure App Configuration) che usa lo `UserId` come seed per la distribuzione.

## Confronto con Java

| Togglz / FF4J | Microsoft.FeatureManagement |
|---|---|
| `Feature` enum + `isActive()` | `IFeatureManager.IsEnabledAsync("name")` |
| `@Toggle(name = "FEATURE")` | `[FeatureGate("FeatureName")]` |
| Feature store in database o file | Configurazione in `appsettings.json` o Azure App Config |
| Rollout percentuale via configurazione | Filtro `Percentage` in `FeatureManagement` config |
| Admin console per toggle runtime | Azure App Configuration portal |
| `FeatureContext.getFeatureManager()` | `IFeatureManager` iniettato via DI |
